PHP的多进程–IPC之信号

正文

PHP 实现进程通信（IPC：InterProcess Communication）的方式有以下几种：共享内存、信号量、消息队列、信号、管道和 socket。

这篇的内容是 PHP 通过信号实现进程通信。使用信号量需要 PHP 安装 pcntl 扩展（在 congfigure 时使用 --enable-pcntl 配置）。

信号

信号由操作系统内核或者是某个进程产生，在满足某个条件时触发，通知另外一个进程执行某个事件（个人感觉类似于操作系统管理进程的中断系统）。在 Windows 下，当我们无法正常结束一个程序时，可以用任务管理器强制结束这个进程，但这其实是怎么实现的呢？ 同样的功能在 Linux 上是通过生成信号和捕获信号来实现的，运行中的进程捕获到这个信号然后作出一定的操作并最终被终止。

当进程接到某个信号时，可以进行以下 3 类操作之一：

第一种是类似中断的处理程序，对于需要处理的信号，进程可以指定处理函数，由该函数来处理；

第二种方法是，忽略某个信号，对该信号不做任何处理，就象未发生过一样（还记得在僵尸进程那篇博客中，父进程对于子进程退出发送给它的 SINCHLD 信号就采用了忽略的行为）；

第三种方法是，对该信号的处理保留系统的默认值，这种缺省操作，对大部分的信号的缺省操作是使得进程终止。

常见的信号及其对目标进程的操作见博客：PHP 信号管理

通过对某个进程发送信号，就实现了进程间的通信。

相关函数

PHP 的 pcntl 扩展中关于信号的函数有以下几个：

// 安装一个信号处理器，在接收到$signal信号时调用$callback函数
pcntl_signal($signal, $callback)

// 向进程号为$pid的进程发送$signal信号
posix_kill($pid, $signal)

// 检测是否有新的信号到达，并调用已安装的信号信号处理器，PHP版本号大于5.3可用
pcntl_singal_dispath()

// 每执行$n条低级指令，就去检测是否有信号进入该进程，效率低，不推荐用（PHP版本低于5.3时用）
declare(ticks=$n)

案例

<?php
$parentpid = posix_getpid();
echo "parent progress pid:{$parentpid}\n";

// 定义一个信号处理函数
function sighandler($signal) {
    if ($signal == SIGINT) {
        $pid = getmypid();
        // 退出并输出对应信息
        exit("{$pid} process, Killed!".PHP_EOL);
    }
}

//php  version < 5.3 .每执行一条低级指令，就检查一次是否出现该信号。效率损耗很大。
//declare(ticks=1);
$child_list = [];

//注册一个信号处理器。当发出该信号的时候对调用已定义的函数
pcntl_signal(SIGINT, 'sighandler');

for($i = 0; $i < 3; $i++) {
    $pid = pcntl_fork();
    if ($pid == 0) {
        // 子进程
        while (true) {
            // 检测信号，调用已安装的信号信号处理器
            pcntl_signal_dispatch();
            echo "I am child: ".getmypid(). " and i am running !".PHP_EOL;
            sleep(rand(1,3));
        }
    }elseif($pid > 0) {
        $child_list[] = $pid;
    }else{
        die('fork fail!'.PHP_EOL);
    }
}

// 延迟5秒后，向各个子进程发SIGINT信号
sleep(5);
foreach ($child_list as $key => $pid) {
    posix_kill($pid, SIGINT);
}

sleep(2);
echo "{$parentpid} parent is end".PHP_EOL;

执行后输出为：

parent progress pid:2752
I am child: 2753 and i am running !
I am child: 2754 and i am running !
I am child: 2755 and i am running !
I am child: 2755 and i am running !
I am child: 2754 and i am running !
I am child: 2753 and i am running !
I am child: 2755 and i am running !
I am child: 2754 and i am running !
I am child: 2755 and i am running !
2753 process, Killed!
2754 process, Killed!
2755 process, Killed!
2752 parent is end

可见，父进程创建了 3 个子进程，并通过对它们发送信号将它们结束，实现了进程间的通信。
