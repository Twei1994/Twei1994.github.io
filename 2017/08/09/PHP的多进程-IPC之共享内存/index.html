正文

PHP 实现进程通信（IPC：InterProcess Communication）的方式有以下几种：共享内存、信号量、消息队列、信号、管道和 socket。

这篇的内容是 PHP 通过共享内存实现进程通信。使用共享内存需要 PHP 安装 shmop 扩展或 semaphore 扩展（在 congfigure 时使用 --enable-shmop 或 --enable-sysvsem 和 --enable-sysvshm）。

共享内存

定义

首先介绍什么是共享内存。在之前关于进程和线程的介绍中，我们知道当操作系统启动一个进程的时候，会给进程分配一块虚拟内存空间，它可以映射到实际的物理内存上。

不同进程的物理内存是彼此独立的。但 Linux/UNIX 系统为了实现进程间的通信，从物理内存中划分出了一部分，称为“共享内存”，顾名思义，就是可以被多个进程所共享的内存，这块内存可以被多个进程读和写，从而实现了进程间的通信。

查看

通过命令 ipcs -m 可以查看系统中的共享内存以及其被几个进程使用。

PHP实现

PHP 有两种创建共享内存的方式，都可以实现进程通信，分别是基于 shmop 扩展的 shmop 系列函数和基于 Semaphore 扩展的 sem 类函数。

shmop系列函数

相关函数介绍：

//Shared Memory Functions

/**
 * shmop_open(int $key , string $flags , int $mode , int $size)
 * $key 共享内存的key
 * $flags 的值有以下几种
 * a :  创建一个只读的共享内存区。
 * c :  如果共享内存区已存在，则打开该共享内存区，并尝试读写。否则新建共享内存区
 * w ： 创建一个读写共享内存区
 * n :  创建一个共享内存区，如果已存在，则返回失败
 *
 * $mode 读写权限。如0755 0644 等
 * $size 申请共享内存区的大小
 */

/**
 * shmop_read( resource $shmid , int $start , int $count)
 * 将从共享内存块中读取数据
 * $shmid 共享内存id，资源类型
 * $start 从共享内存的那个字节开始读起
 * $count 一次读取多少个字节。
 * 如果count值小于发送的信息长度，则信息会被截断。
 */

/**
 * shmop_write(resource $shmid , string $data , int $offset)
 * 将数据写入共享内存块
 * $data 将要写入的数据
 * $offset 从共享内存块的那个位置开始写入。
 * 该函数的返回值是写入数据的长度。
 */

/**
 * shmop_size(resource $shmid);
 * 返回当前共享内存块，已经使用的大小
 */


/**
 * shmop_delete ( resource $shmid )
 * 删除一个共享内存块的,删除引用关系
 */

/**
 * shmop_close ( resource $shmid )
 * 关闭共享内存块
 * 要先使用shmop_delete 之后才能继续使用shmop_close
 */

有两点需要注意：

  1. shmop_read 函数第 2 个参数是读取的起始位置，第3个参数是要读取的长度，如果你要读取的长度小于信息长度，原信息会被截断成你指定的长度；

  2. shmop_write 函数仅可写字符串内容，所以可以使用 pack() 函数将数据转成二进制字符串后写入共享内存（见实例2）；

实例1–建立、写入和读取共享内存（一个进程内完成）

<?php
// ftok() 函数将一个文件和描述符转成一个唯一的id
$shm_key = ftok(__FILE__, 't');
$shm_id = shmop_open($shm_key, "c", 0655, 1024);
shmop_write($shm_id, "Hello world".PHP_EOL, 0);
$data = shmop_read($shm_id, 0, 100);
var_dump($data);
shmop_delete($shm_id);
shmop_close($shm_id);
?>

执行后输出我们写入的字符串 Hello world。

实例2–两个进程，一个向共享内存写数据，一个读取共享内存的数据

write.php 用于写入数据，它的代码如下：

<?php  
$key = 0x4337b700;  
$size = 4096;  
$shmid = @shmop_open($key, 'c', 0644, $size);  
if($shmid === FALSE){  
    exit('shmop_open error!');  
}  

$data = 'Hello Wrold';  

$length = shmop_write($shmid, pack('a*',$data), 0);  
if($length === FALSE){  
    exit('shmop_write error!');  
}  

@shmop_close($shmid);  

exit('succ');  
?>

运行 write.php ，输出 succ，执行 ipcs -m 可以看到系统中有一个 key 为 0x4337b700 的共享内存。

read.php 用于写入数据，它的代码如下：

<?php   
$key = 0x4337b700;  
$size = 256;  
$shmid = @shmop_open($key, 'w', 0644, $size);  
if($shmid === FALSE){  
    exit('shmop_open error!');  
}  

$data = unpack('a*', shmop_read($shmid, 0, 256));  
if($data === FALSE){  
    exit('shmop_read error!');  
}
// 不加shmop_delete，仅仅关闭共享内存块，并没有删除
//shmop_delete($shmid);  
@shmop_close($shmid);  

exit($data[1]);  
?>

执行后输出： Hello Wrold，说明我们通过另外一个进程获取到了之前进程的变量，实现了进程间的通信。

sem系列函数

sem 系列创建的共享内存类似 key-value 格式，相关函数介绍：

// 创建一个共享内存
$shm_id = shm_attach($key, 1024, 0666); 

// 设置一个值
$shar_key = 1;
shm_put_var($shm_id, $shar_key, 'test');

// 删除一个 key
shm_remove_var($shm_id, $shar_key);

// 获取一个值
$value = shm_get_var($shm_id,  $shar_key);

// 检测一个 key 是否存在
shm_has_var($shm_id,  $shar_key);

// 从系统中移除一个共享内存
shm_remove($shm_id);

// 关闭和共享内存的连接
shm_detach($shm_id);

需要注意的地方：

  1. shm_put_var 第三个参数 写入的值 是一个混合类型，所以没有shmop_write的局限性；

  2. $shar_key 只能是 int 型的参数。

实例1–建立、写入和读取共享内存（一个进程内完成）

<?php
// Get the file token key
$key = ftok(__FILE__, 'a');
$shar_key = 1;

// 创建一个共享内存，key，长度，权限
$shm_id = shm_attach($key, 1024, 0666);
if ($shm_id === false) {
    die('Unable to create the shared memory segment' . PHP_EOL);
}

// 设置一个值
shm_put_var($shm_id, $shar_key, 'test');

// 删除一个key
//shm_remove_var($shm_id, $shar_key);

// 获取一个值
$value = shm_get_var($shm_id,  $shar_key);
var_dump($value);

// 检测一个key是否存在
// var_dump(shm_has_var($shm_id,  $shar_key));

// 从系统中移除
shm_remove($shm_id);

// 关闭和共享内存的连接
shm_detach($shm_id);

执行后输出 test。

实例2–多进程实现进程通信，模拟实际业务

<?php
// 创建共享内存区域
$shm_key = ftok(__FILE__, 't');
$shm_id = shm_attach( $shm_key, 1024, 0655 );
const SHARE_KEY = 1;
$childList = [];

// 开3个进程，读写该内存区域
for( $i = 0; $i < 3; $i++ ){
    $pid = pcntl_fork();
    if( $pid == -1 ){
        exit('fork fail!' . PHP_EOL);
    }elseif( $pid == 0 ){
        // 子进程从共享内存块中读取，+1后写回
        // 模拟业务处理逻辑延迟
        $sec = rand(1, 3);
        sleep($sec);
        if (shm_has_var($shm_id, SHARE_KEY) ){
            // 有值，加一
            $count = shm_get_var($shm_id, SHARE_KEY);
            $count++;
            // 模拟业务处理逻辑延迟
            $sec = rand(1, 3);
            sleep($sec);
            // 写入共享内存中
            shm_put_var($shm_id, SHARE_KEY, $count);
        }else{
            // 无值，初始化
            $count = 0;
            // 模拟业务处理逻辑延迟
            $sec = rand( 1, 3 );
            sleep($sec);
            // 写入共享内存中
            shm_put_var($shm_id, SHARE_KEY, $count);
        }

        echo "child process " . getmypid() . " is writing ! now count is $count\n";

        exit( "child process " . getmypid() . " end!\n" );
    }else{
        // 主进程记录子进程号，用于回收
        $childList[$pid] = 1;
    }
}

// 等待所有子进程结束
while( !empty($childList)){
    // 返回结束子进程号
    $childPid = pcntl_wait($status);
    if ($childPid > 0){
        unset( $childList[$childPid] );
    }
}

// 父进程读取最终共享内存中的值
$count = shm_get_var($shm_id, SHARE_KEY);
echo "final count is " . $count . PHP_EOL;

// 从系统中移除共享内存
shm_remove($shm_id);
// 关闭和共享内存的连接
shm_detach($shm_id);

上面代码的逻辑很简单，创建 3 个子进程，对同一个共享内存中的数据进行读写。有一个 count 的值，如果读到就 +1，下面我们看一下运行结果：

我们发现多次运行的结果不同，但都不是我们想要的从 0 加到 2。这是为什么呢？

简单分析一下，不难发现，当我们开启创建进程的时候，3 个子进程可能同时打开了共享内存区域，此时他们几乎是同步的，所以读到的信息都是没有 count 值，此时他们执行自己的业务逻辑然后将 count 设为 0 并写入了内存区域，也就是说，共享内存是没有锁机制的。

这并不是我们想要的结果，三个子进程互相抢占了资源，这是不合理的，那怎么规避这个问题呢？答案是通过信号量！
