正文

之前介绍过，每次调用 fork 函数，操作系统就会产生一个子进程，子进程所有的堆栈信息都是原封不动复制父进程的，而在 fork 之后，父进程与子进程实际上是相互独立的，父子进程不会相互影响。

也就是说，fork 调用位置之前的所有变量，父进程和子进程是一样的，但 fork 之后则取决于各自的动作，且数据也是独立的；因为数据已经完整的复制给了子进程。而唯一能够区分父子进程的方法就是 fork 函数的返回值。

实例1–多进程写入文件（错误示范）

<?php
define('FORK_NUMS', 3);
$pids = array();
$fp = fopen('./test.log', 'wb');
$num = 1;

for($i = 0; $i < FORK_NUMS; ++$i) {

    $pids[$i] = pcntl_fork();
    if($pids[$i] == -1) {
        die('fork error');
    }elseif($pids[$i]) {
    }else{
        for($i = 0; $i < 5; ++$i) {
             // 给文件加锁
            flock($fp, LOCK_EX);
            fwrite($fp, getmypid() . ' : ' . date('Y-m-d H:i:s') . " : {$num} \r\n");
             // 文件解锁
            flock($fp, LOCK_UN);
            echo getmypid(), ": success \r\n";
            ++$num;
        }
        exit;
    }
}

foreach($pids as $k => $v) {
    if($v) {
        pcntl_waitpid($v, $status);
    }
}

fclose($fp);
?>

上面的代码实现了这个功能：创建了三个子进程，向 test.log 文件写入内容，test.log 内容如下：

我们可以看到三个子进程的 pid，它们分别执行了 5 次，时间几乎是在同时。但是 $num 的值并没像我们期望的那样从 1－15 进行递增。子进程中的变量是各自独立的，互不影响。子进程会自动复制父进程空间里的变量。

实例2–多进程写入文件（共享数据）

如何在进程中共享数据？我们通过 PHP 的共享内存函数 shmop 来实现（本质上是 PHP 进程通信的一种方式）。

<?php
define('FORK_NUMS', 3);
$pids = array();
$fp = fopen('./test.log', 'wb');
$num = 1;

//共享内存段的key
$shmKey = 123;
//创建共享内存段
$shmId = shmop_open($shmKey, 'c', 0777, 64);
//写入数据到共享内存段
shmop_write($shmId, $num, 0);

for($i = 0; $i < FORK_NUMS; ++$i) {
    $pids[$i] = pcntl_fork();
    if($pids[$i] == -1) {
        die('fork error');
    } else if ($pids[$i]) {
        //阻塞，等待子进程退出
        //注意这里，如果是非阻塞的话，$num的计数会出现问题。
        pcntl_waitpid($pids[$i], $status);
    } else {
        //读取共享内存段中的数据
        $num = shmop_read($shmId, 0, 64);

        for($i = 0; $i < 5; ++$i) {
            fwrite($fp, getmypid() . ' : ' . date('Y-m-d H:i:s') . " : {$num} \r\n");
            echo getmypid(), ": success \r\n";
            //递增$num
            $num = intval($num) + 1;
        }

        //写入到共享内存段中
        shmop_write($shmId, $num, 0);
        exit;
    }
}
//shmop_delete不会实际删除该内存段，它将该内存段标记为删除。
shmop_delete($shmId);
shmop_close($shmId);
fclose($fp);
?>

上面的代码执行后，test.log 的内容为：

11392 : 2017-08-08 14:32:59 : 2 
11392 : 2017-08-08 14:32:59 : 3 
11392 : 2017-08-08 14:32:59 : 4 
11392 : 2017-08-08 14:32:59 : 5 
11393 : 2017-08-08 14:32:59 : 6 
11393 : 2017-08-08 14:32:59 : 7 
11393 : 2017-08-08 14:32:59 : 8 
11393 : 2017-08-08 14:32:59 : 9 
11393 : 2017-08-08 14:32:59 : 10 
11394 : 2017-08-08 14:32:59 : 11 
11394 : 2017-08-08 14:32:59 : 12 
11394 : 2017-08-08 14:32:59 : 13 
11394 : 2017-08-08 14:32:59 : 14 
11394 : 2017-08-08 14:32:59 : 15

这样就实现了多进程之间共享数据 $num。
