PHP的多进程--基础介绍

正文

PHP 的设计目的就是用来快速地开发 WEB 应用，这也注定了它在某些方面的先天不足，例如在 CLI 环境下处理大量数据的情况，或者在并发编程方面，都显得力不从心。

虽然 PHP 本身不支持多进程，但基于 Linux 的 PHP 扩展 PCNTL 却可以提供多进程编程，大大提高了 PHP 的效率，可以让 PHP 实现一些复杂度高的任务。

注：PHP 的多进程是在 CLI 下执行的，不适合在 Web 服务器上处理请求。

PCNTL

简介

pcntl 是用于实现 PHP 多进程编程的扩展，只能在 Linux/Unix 下安装，所以不能通过它实现 Windows 下的多进程。

安装

有以下两种安装方式，根据 PHP 的安装方式来选择：

  • 重新编译安装 PHP

重新编译安装 PHP，在 configure 时，加入 --enable-pcntl 配置来打开并启动 pcntl 扩展；

  • 通过phpize

如果不想重新编译安装 PHP，可以利用 phpize 来动态安装，分为以下几步：

  1. 通过执行 phpize 来检查是否安装了 phpize，如果没安装则执行 yum install php_devel 来安装 phpize 指令，并记录它的目录（如 phpize 在我这里的目录位于 /usr/local/php/bin/phpize）；
  2. 去 php.net 下载相应版本的 php 源文件；
  3. 解压源文件，并进入 pcntl 扩展目录下 cd /usr/local/php/ext/pcntl；
  4. 执行 phpize 命令 /usr/local/php/bin/phpize，执行完会生成 configure 文件；
  5. 执行 ./configure --with-php-config=/usr/local/php/bin/php-config，其中 /usr/local/php 是 PHP 的安装目录；
  6. 编译和安装 make && make install，执行成功后会在 PHP 扩展的目录下生成一个 pcntl.so 的文件；
  7. 编辑 PHP 配置文件 /usr/local/php/etc/php.ini，加入 extension=pcntl.so;
  8. 重启 PHP 环境，重启 Web 服务器；

查看

  • 通过 php -m 可以查看 PHP 当前应用的扩展，里面有 pcntl 说明该扩展已经应用；

  • 也可以用 php -r "echo pcntl_fork();" 来执行 pcntl 扩展提供的函数来查看扩展是否成功启动，返回一个正整数说明成功；

实战

安装好了扩展，终于可以实践了。先给出最重要的一个函数：

pcntl_fork

pcntl_fork() 函数创建一个子进程，这个子进程仅有 PID（进程号）和 PPID（父进程号）与其父进程不同，其他的都相同。

返回值：成功时，在父进程执行线程内返回产生的子进程的 PID，在子进程执行线程内返回 0。失败时，在父进程上下文返回 -1，不会创建子进程，并且会引发一个 PHP 错误。

实例1–创建单个子进程

<?php
// 创建子进程
$pid = pcntl_fork();
if($pid == -1){
    // 返回值为-1，创建失败
    die('could not fork');
}elseif($pid){
    // 返回值大于0，是父进程
    echo "parent \t", date("H:i:s", time()), "\n";
}else{
    // 返回值等于0，是子进程
    echo "child \t", date("H:i:s", time()), "\n";
}
?>

如果通过 pcntl_fork 创建子进程成功的话，系统就有了2个进程，一个为父进程，一个为子进程，子进程的 id 号为 $pid。

当系统运行到 $pid = pcntl_fork(); 时，会这个地方进行分支，父子进程各自开始运行各自的程序代码。

上面的代码，输出结果为：

parent     08:49:18
child     08:49:18

也许你很奇怪，为什么一个 if 和 else 互斥的代码中，都输出了结果？

这恰恰说明了有两个进程在执行后面的代码。父进程的变量 $pid 值大于 0，所以执行 elseif 中的代码，子进程的变量 $pid 值等于 0，就会执行 else 中的代码。

至于谁先谁后的问题，这得要看系统资源的分配了。

实例2–创建多个子进程

当我们希望创建多个子进程时，可以使用循环，在循环内部使用 pcntl_fork 函数。

<?php
// 创建子进程
for ($i = 0; $i < 2; $i++) { 
    $pid = pcntl_fork();
    if($pid == -1){
        // 返回值为-1，创建失败
        die('could not fork');
    }elseif($pid){
        // 返回值大于0，是父进程
        echo "parent \t", getmypid(), "\t", date("H:i:s", time()), "\n";
    }else{
        // 返回值等于0，是子进程
        echo "child \t", getmypid(), "\t", date("H:i:s", time()), "\n";
    }
}
?>

通过 getmypid() 函数可以得到当前进程的进程 id（pid），可以帮助我们查看进程的个数。

上面的代码循环的次数为 2，所以创建了 2 个子进程。如果你是这样想的，那就大错特错了。它的执行结果是这样的：

parent     7221    09:03:41
parent     7221    09:03:41
child     7223    09:03:41
child     7222    09:03:41
parent     7222    09:03:41
child     7224    09:03:41

从结果可以看到，这段代码执行时一共产生了 4 个不同的进程，其中一个（7222）既是父进程，又是子进程。和我们想的一个父进程加 2 个子进程相差比较大。

这是为什么呢？还是没有深刻理解子进程的概念。

当 fork 出来子进程后，它就与父进程完成独立了，并保持着之前的执行状态。从上面的结果可以推断出，当（$i == 0）第一次循环时，父进程为 7221，通过 fork 它得到了一个子进程 7222，它们分别执行各自的代码，输出 parent 7221 09:03:41 和 child 7222 09:03:41；然后 7221 进程继续执行第二次循环，同时，7222 进程也会执行第二次循环，因为它的执行状态和父进程是一样的。它们作为父进程 fork 出来了 7223 和 7224 两个子进程，从而又输出了 4 条结果。

实例3–创建多个子进程

理解了原因之后，我们可以得到上面方法的一个缺点：循环未结束，子进程会不停的创建子进程，可能造成系统资源被耗尽的情况。

所以，一般子进程代码运行完成后，要加入 exit 来确保子进程正常退出。

那么上面的代码可以修改成这样的：

<?php
for ($i=0; $i < 2; $i++) {
    $pid = pcntl_fork();
    if($pid == -1){
        // 返回值为-1，创建失败
        die('could not fork');
    }elseif($pid){
        // 返回值大于0，是父进程
        echo "parent \t",posix_getppid(),"\t", getmypid(), "\t", date("H:i:s", time()), "\n";
    }else{
        // 返回值等于0，是子进程
        echo "child \t",posix_getppid(),"\t", getmypid(), "\t", date("H:i:s", time()), "\n";
        exit;
    }
}

通过 posix_getppid() 可以得到当前进程的父进程，我们把它放在输出中。这次的输出结果是：

parent     2366    7470    09:35:48
parent     2366    7470    09:35:48
child     7470    7472    09:35:48
child     7470    7471    09:35:48

可以看到，这次只有 3 个进程，每个循环中， 父进程 7470 都创建了一个子进程。在实际项目中，要采用这种方式，即在子进程的执行代码最后添加 exit，而不是实例 2 的形式。
