牛客网PHP工程师能力评估

前言

今天发现牛客网还有一个 PHP 工程师能力评估，就顺手做了一下。这里记录下做错的和不太确定的题。

正文

unset 和 null

题目：下面代码的输出结果是什么？

$a = "hello";
$b = &$a;
unset($b);
$b = "world";
echo $a;

答案： hello

分析：

$a = "hello" 这句将字符串 hello 赋值给变量 a，实际上是建立了一个 zval，并将 a 存到符号表中，同时建立了这两者之间的联系。

$b = &$a 这句声明变量 b 是 a 引用，本质上是将 b 存到符号表中，同时将它指向 a 对应的那个 zval。 这样，不管 a 和 b 哪个发生变化，都会同时改变另一个的值。

unset($b) 的作用是将变量值 b 从符号表中删除，也就断开了它和 zval 之间的联系。这个时候，因为还有 a 指向这个 zval，所以它不算泄露的内存，也就不会被回收。

$b = "world" 相当于新建了一个 zval 同将符号表中的 b 指向它，这个操作不会对 a 产生任何影响。

所以最终输出 a，就是原来的值。

empty()

题目：下面代码的输出结果是什么？

<?php
        $var = FALSE;
        if (empty($var)){
               echo "null";
        }else{
               echo "have value";
        }
?>

答案：null

分析：

如果 var 是非空或非零的值，则 empty() 返回 FALSE。

换句话说，””、 0、 “0”、 NULL、 FALSE、 array()、 var $var、以及没有任何属性的对象都将被认为是空的，如果 var 为空，则返回 TRUE。

strcmp()

题目：下面代码的输出结果是什么

$str = “LAMP”;
$str1 = “LAMPBrother”;
$strc = strcmp($str,$str1);
switch ($strc){
    case 1:
        echo“str > str1”;
        break;
    case –1:
        echo“str < str1”;
        break;
    case 0:
        echo“str=str1”;
        break;
    default:
        echo“str <> str1”;
}

答案： str <> str1

分析：

函数 strcmp($str1, $str2)，如果 $str1 和 $str2 存在包含关系，并且是从头开始就是包含关系，（类似题目中的情况），但 ‘AB’ 和 ‘cAB’不算，这时返回的就是它们不同的字符的个数，谁的字符多谁大；如果不存在这种包含关系，那么就根据不同字符的ASCII码大小关系返回-1,1。

strcmp("A", "Abcdf")   // -4，存在从头包含关系
strcmp("a", "Abcdf")   // 1
strcmp("abc", "Aabcd") // 1，包含，但是从头开始

三元运算符

题目：哪一个三元运算符相当于此脚本？

<?php
       if ($a < 10){
              if($b > 11){
                     if($c == 10 && $d != $c) {
                            $x=0;
                     }else {
                            $x=1;
                     }
              }
       }
?>

答案：没有三元运算符能表示

分析：三元运算符是水平层级的运算，而多层 if 嵌套是一种递进的运算。也就是不管三元运算符前面的条件是什么，$x = (.......) ? 0: 1，最终只有两个结果， x = 0 或者 x = 1。但是在多层嵌套的 if 语句中，当 a >= 10 时，这个程序已经结束，不会对 x 进行赋值。

final修饰符

题目：在PHP面向对象中，下面关于 final 修饰符描述错误的是（ ）

  • A. 使用 final 标识的类不能被继承
  • B. 在类中使用 final 标识的成员方法，在子类中不能被覆盖
  • C. 不能使用 final 标识成员属性
  • D. 使用 final 标识的成员属性，不能在子类中再次定义

答案：D，不能用 final 修饰成员属性

分析：

final 修饰符，只能用于修饰类和方法。

修饰类的时候，该类不能被继承；修饰方法的时候，该方法不能被子类重写。

对象的复制

题目：下面代码的输出结果是什么？

<?php 
class A{
    public $num=100; 
}
$a = new A();
$b = $a;
$a->num=200;
echo $b->num;
?>

答案：200

分析：

知识点是对象的复制。在 PHP 中，对象是 $a -> 对象标识符 -> 实际数据 这种方式存储的。

有 3 种实现对象复制的方法：

$b = $a;  
$b = &$a;
$b = clone $a;

第一种方式，$b 复制了 $a 的对象标识符，但指向同一个实际数据，修改对象实际数据时有影响。但修改 $b 的值时，如 $b = 'aaa'，会写时复制，就成了两个独立的变量了；

第二种方式，$b 是 $a 的引用，它们共享一个对象标识符，共享同一个内存地址，所以就相当于一个变量，任何更改都会影响到对方；

第三种方式，采用 clone 实现对象的复制，相当于新建了一个对象，不管是对象标示符还是实际数据都是独立的，更改不会影响对方。

get_class()，获取对象类名

题目：获得实例化对象所属类名字的函数是（）

答案：get_class()

分析：

相关的函数有 3 个：

get_class(object $object)  // 返回实例所属的类名
get_class_vars(string $class_name)  // 根据类名返回类的属性组成的数组
get_class_methods(类名/对象)   // 根据类名或对象返回类的方法组成的数组

注：get_class_vars(get_class($object)) 通过对象获得实例的属性

PDO错误模式

PDO中一共提供了三种不同的错误处理模式

  • PDO::ERRMODE_SILENT：不显示错误信息，只显示错误码
  • PDO::ERRMODE_WARNING：显示警告错误
  • PDO::ERRMODE_EXCEPTION：抛出异常
