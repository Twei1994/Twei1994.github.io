PHP的多进程–IPC之管道

正文

PHP 实现进程通信（IPC：InterProcess Communication）的方式有以下几种：共享内存、信号量、消息队列、信号、管道和 socket。

这篇的内容是 PHP 通过管道实现进程通信。使用管道需要 PHP 安装 posix 扩展（默认安装）。

管道

我对管道的理解是采用消息队列方式操作的文件。

为什么说管道是文件呢？因为它的本质是一段系统内核的缓冲区，可以看做是一个伪文件。在我们使用管道时，需要创建、打开、读、写和关闭，就和我们操作文件一样。

为什么说管道是采用消息队列的方式呢？因为它实际上的数据结构是一个环形队列。不同的线程都可以向里面写，也可以从里面读。写在队列末尾，读就是从队列头部删除。

管道分为两种，匿名管道（pipe）和命名管道（FIFO）。匿名管道用于父子进程通信，而命名管道可以用于任意两个进程通信。

相关函数

// 创建一个命名管道，参数为文件路径加权限
posix_mkfifo($pipe_path,0664)

// 以写入的方式打开命名管道，用于写数据
fopen($pipe_path,'w')

// 以读取的方式打开命名管道，用于读数据
fopen($pipe_path,'r')

// 向命名管道写入数据，参数为文件句柄加数据
fwrite($file,'hello world');

// 从命名管道中读取指定字节长度的数据
fread($file,20);

// 删除一个命名管道 
unlink($file)

实例

<?php
// 定义管道路径,与创建管道
$pipe_path = '/data/test.pipe';
if(!file_exists($pipe_path)){
    if(!posix_mkfifo($pipe_path,0664)){
        exit("create pipe error!");
    }
}
$pid = pcntl_fork();
if($pid == 0){
    // 子进程,向管道写数据
    $file = fopen($pipe_path,'w');
    while (true){
        fwrite($file,'hello world');
        $rand = rand(1,3);
        sleep($rand);
    }
    exit('child end!');
}else{
    // 父进程,从管道读数据
    $file = fopen($pipe_path,'r');
    while (true){
        $rel = fread($file,20);
        echo "{$rel}\n";
        $rand = rand(1,2);
        sleep($rand);
    }
}

上面的代码实现了一个子进程和一个父进程。它们两个都是一直循环，子进程不断地向管道中写入 hello world 这个字符串，父进程不断地从管道中取出 20 字节长度的数据并显示。

执行结果：

hello worldhello wor
ldhello worldhello w
orldhello worldhello
 worldhello worldhel
lo worldhello worldh

可见每次都是读取 20 字节的内容。
