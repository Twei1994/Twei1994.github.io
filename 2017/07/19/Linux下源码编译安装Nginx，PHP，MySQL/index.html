Linux下源码编译安装Nginx，PHP，MySQL

前言

就我个人而言，用 PHP 做 WEB 开发，过程一般是这样的：先在 Windows 下进行开发，然后再部署到服务器上，所以在 Windows 和 Linux 下安装 PHP 环境的技能都要掌握。

Windows 下服务器通常采用的是 Apache，而在 Linux 下通常会使用 Nginx，因为它在支持高并发请求的同时也能保持高效的服务。

在之前的一篇博客中，讲了如何在 Windows 环境下安装 PHP7，Apache 和 MySQL，在这里再讲一下如何通过源码编译的方式在 Linux 下安装 PHP 和 Nginx。

安装环境

  • 我的操作系统是 Win7 下的虚拟机： Ubuntu 16.04
  • Nginx版本：1.12.1
  • PHP版本： 7.1.7

安装过程

安装路径

  • 源码路径：/usr/local/src，下载下来的源码都放在这里
  • 安装路径：/usr/local，编译后软件安装在这里

下载源码包

下载地址：

  • Nignx：http://nginx.org/en/download.html
  • PHP：http://php.net/downloads.php

更新系统目前软件包

#更新本地仓库的版本信息
sudo apt-get update
#升级软件包
sudo apt-get upgrade

Nginx

下载安装依赖包

在安装 Nginx 之前，要把它依赖的软件装好，包括 openssl，pcre 和 zlib。

安装 openssl

Ubuntu 16.04 中自带了 openssl，无需安装。

如果没有安装，可以通过以下的方式安装：

#
#----------------------------------------
# 下载地址： http://www.openssl.org/
#----------------------------------------

#进入源码目录
cd /usr/local/src

#解压
tar -zxv -f openssl-1.1.0f.tar.gz

#进入源码目录
cd openssl-1.1.0f

#配置
./config --prefix=/usr/local/openssl --openssldir=/usr/local/openssl/conf

#编译安装
make && make install

#检验安装
/usr/local/openssl/bin/openssl version -a

安装 pcre

#
#----------------------------------------
#下载地址： http://www.pcre.org/
#----------------------------------------

#进入源码目录
cd /usr/local/src

#解压
tar -zxv -f pcre-8.40.tar.gz

#进入源码目录
cd pcre-8.40 

#执行配置
./configure --prefix=/usr/local/pcre/

#编译安装
make && make install

安装 zlib

#
#----------------------------------------
#下载地址： http://www.zlib.net/
#----------------------------------------

#进入源码目录
cd /usr/local/src

#解压
tar -zxv -f zlib-1.2.11.tar.gz

#进入源码目录
cd zlib-1.2.11

#配置
./configure --prefix=/usr/local/zlib/

# 编译安装
make && make install

安装 Nginx

安装好了 openssl，pcre 和 zlib 之后，终于可以安装 Nginx了。

#添加www用户和组
groupadd www
useradd -g www www    

#创建网站根目录
mkdir -p     /var/www/
chmod -R 775 /var/www/

#进入源码目录
cd /usr/local/src

#解压
tar -zxvf nginx-1.12.1.tar.gz

#进入安装目录
cd nginx-1.12.1

#配置 (使用openssl、pcre、zlib的源码路径)
./configure \
--user=www \
--group=www \
--prefix=/usr/local/nginx \
--with-http_ssl_module \
--with-openssl=/usr/local/src/openssl-1.1.0f \
--with-pcre=/usr/local/src/pcre-8.40 \
--with-zlib=/usr/local/src/zlib-1.2.11 \
--with-http_stub_status_module \
--with-threads

#编译安装
make && make install

这样 Nginx 就安装好了，可以通过下面的指令进行验证。

#验证
/usr/local/nginx/sbin/nginx -V

启动并测试

通过以下的指令启动 Nginx：

#启动
/usr/local/nginx/sbin/nginx

启动后，可以下面两种方式查看 Nginx 是否启动：

#测试端口 
netstat –na|grep 80

说明 Nginx 已经开始监听 80 端口了。

#浏览器中测试 
http://localhost

打开浏览器后，输入 http://localhost，可以发现这个页面，说明 nginx 已经安装好并开始工作了。

PHP

下载安装依赖包

PHP 需要的依赖包比较多，包括之后常用的一些拓展，都需要在 configure 前下载下来，具体的有 libxml2-dev，libssl-dev，curl，libcurl4-gnutls-dev，libjpeg-dev，libpng-dev，libmcrypt-dev，libreadline6，libreadline6-dev，freetype，freetype-dev，libxslt，libxslt-dev，bzip2。

通过 sudo apt-get install + 软件包名 来下载和安装。

以上这样基本就足够了，如果在之后有需要的，再下载安装即可。

安装 PHP

#进入源码目录
cd /usr/local/src

#解压
tar -zxvf php-7.1.1.tar.gz

#进入安装目录
cd php-7.1.7

#配置（如果报错并停止，看下文的注意部分）
./configure --prefix=/usr/local/php --with-curl --with-freetype-dir --with-gd --with-gettext --with-iconv-dir --with-kerberos --with-libdir=lib64 --with-libxml-dir --with-mysqli --with-openssl --with-pcre-regex --with-pdo-mysql --with-pdo-sqlite --with-pear --with-png-dir --with-jpeg-dir --with-xmlrpc --with-xsl --with-zlib --with-bz2 --with-mhash --enable-fpm --enable-bcmath --enable-libxml --enable-inline-optimization --enable-gd-native-ttf --enable-mbregex --enable-mbstring --enable-opcache --enable-pcntl --enable-shmop --enable-soap --enable-sockets --enable-sysvsem --enable-sysvshm --enable-xml --enable-zip

#编译（编译时间比较长）
make

#安装
make install

验证

经过以上步骤，就在 Ubuntu 上安装好了 PHP，可以通过以下方式验证：/usr/local/php/bin/php -v

安装后的配置

此时，虽然我们装好了 Nginx 和 PHP，但它们之间并没有关系。Nginx 可以监听 HTTP 的 80 端口，但是它并不会将 .php 的请求交给 PHP 来解析。我们需要对它们进行配置从而把它们关联起来。

PHP

假设当前路径还在 /usr/local/src/php-7.1.7，执行以下命令：

#创建php的配置文件
cp php.ini-development /usr/local/php/lib/php.ini

然后设置 php.ini，使用： vim /usr/local/php/lib/php.ini 打开 php.ini，找到 cgi.fix_pathinfo 配置项，这一项默认被注释并且值为 1（根据官方文档的说明，这里为了当文件不存在时，阻止 Nginx 将请求发送到后端的 PHP-FPM 模块，从而避免恶意脚本注入的攻击），所以此项应该去掉注释并设置为 0

之前在安装 Nginx 时，增加了 www 用户和用户组，我们需要把它加入 php-fmp 的配置中。

#创建 php-fpm 的配置文件
cp /usr/local/php/etc/php-fpm.conf.default /usr/local/php/etc/php-fpm.conf

在 php-fpm.conf 的最后，可以看到这样一句：include=/usr/local/php/etc/php-fpm.d/*.conf，也就是它引入了 /php-fpm.d/ 下的所有 .conf 文件。（如果在 configure 时没有添加 –prefix，这是是 NONE，需要自己改成 php 的安装路径）

默认情况下 etc/php-fpm.d/ 下有一个名为 www.conf.defalut 的配置用户的文件，执行下面命令复制一个新文件并且打开：

#创建 php-fpm 用户和用户组的文件
cp /usr/local/etc/php-fpm.d/www.conf.default /usr/local/etc/php-fpm.d/www.conf
vim /usr/local/etc/php-fpm.d/www.conf

默认 user 和 group 的设置为 nobody，将其改为 www

Nginx

执行 vim /usr/local/nginx/conf/nginx.conf 编辑nginx配置文件，主要有3处需要修改的地方：

  • 修改配置文件的第一行，默认是 #user nobody，需要去掉注释并改为 user www 或者 user www www 表示 nginx 服务器的权限为 www

  • 修改 nginx 的 server { } 配置块中的内容，修改 location 块，追加 index.php 让 nginx 服务器默认支持 index.php 为首页

  • 然后配置 .php 请求被传送到后端的 php-fpm 模块，默认情况下 php 配置块是被注释的，此时去掉注释并修改为以下内容：

大部分都是默认的，root 是配置 php 程序放置的根目录，是我们在安装 Nginx 时新建的。另外修改的就是 fastcgi_param 中的 /scripts 为 $document_root

启动并验证

之前已经启动了 Nginx，在启动 php-fpm 后，就可以识别 .php 请求了。

在修改了 nginx.conf 后，需要重启 nginx

/usr/local/nginx/sbin/nginx -s stop
/usr/local/nginx/sbin/nginx

启动 php-fpm

/usr/local/php/sbin/php-fpm

启动完毕之后，php-fpm 服务默认使用 9000 端口，使用 netstat -tln | grep 9000 可以查看端口使用情况：

9000 端口正常使用，说明 php-fpm 服务启动成功

最后编辑一个测试的 php 程序，在 Nginx 下的 /var/www/ 目录下创建 index.php 文件，打印一下 php 配置：

<?php
    phpinfo();
?>

　
然后打开浏览器输入 http://localhost/index.php，看到输出页面，说明 Nginx 和 php 都配置成功了：

注意

  1. 为了避免不必要的错误，建议所有的命令都用 root 权限执行，也就是在前面用 sudo;
  2. 在安装 PHP 的配置（configure）阶段，如果依赖包不足，会报错并停止，只要把对应的依赖包装好就行，可以参考我的这篇博客：PHP源码安装过程中的常见问题；
