Apache Rewrite服务器变量

前言

Rewrite 作为 Web 服务器的核心功能之一，提供了通过正则表达式与请求路径的匹配，同时还提供了一个系统级变量供调用，本篇文章主要介绍 Apache 中 Rewrite 模块的服务器变量。

正文

Apache 提供给 rewirte 模块的环境变量大概分成 5个类型。下面一一说明，来源于 apache 官方文档。

其中，经常被用到的有如下几个： HTTP_HOST、HTTP_REFERER、SERVER_NAME、

HTTP headers 部分参数

HTTP_USER_AGENT

含义：用户使用的浏览器（包括版本号），操作系统（包括版本号）。

样例参考值: Mozilla/5.0 (Windows; U; Windows NT 5.1; zh-CN; rv:1.9.0.8) Gecko/2009032609 Firefox/3.0.8

说明: 相当于PHP中的服务器参数: $_SERVER[“HTTP_USER_AGENT”]

HTTP_REFERER

含义：之前网页的地址，告诉服务器当前请求是从哪个页面链接过来的，服务器基此可以获得一些信息用于处理，可以用于防止盗链。

样例参考值: http://www.test.cn/test.php

说明: 相当于PHP中的服务器参数: $_SERVER[“HTTP_REFERER”]

HTTP_COOKIE

含义：随请求传过来的 cookie

样例参考值: ZDEDebuggerPresent=php,phtml,php3

说明: 相当于PHP中的服务器参数: $_SERVER[“HTTP_COOKIE”]

HTTP_FORWARDED

含义：用来表示 HTTP 请求端的真实 IP，如果使用代理则为每一级代理设备的 IP，可以被伪造

样例参考值: client，proxy1，proxy2

说明: 相当于PHP中的服务器参数: $_SERVER[“HTTP_X_FORWARDED_HOST”]

HTTP_HOST

含义：请求中的域名或IP地址，如果端口不为80，则会加上端口号。

样例参考值：如请求为 www.test.com/index.php，HTTP_HOST 值为 www.test.com

说明: 相当于PHP中的服务器参数: $_SERVER[“HTTP_HOST”]

HTTP_PROXY_CONNECTION

含义：使用代理服务器时会产生，对应的非代理项为 HTTP_CONNECTION。

样例参考值: 网络连接代理方面的信息。

HTTP_ACCEPT

含义：客户端希望接受的数据类型。

样例参考值: text/html,application/xhtml+xml,application/xml;

说明: 相当于PHP中的服务器参数: $_SERVER[“HTTP_ACCEPT”]

server internals 部分参数

DOCUMENT_ROOT

含义：当前运行脚本所在的文档根目录，就是 Apache 配置时写在 <DocumentRoot> 中的值

样例参考值: D:/web/www

说明: 相当于PHP中的服务器参数: $_SERVER[“DOCUMENT_ROOT”]

SERVER_ADMIN

含义：配置中 ServerAdmin 的值，服务器管理员邮箱。

样例参考值: 默认值为 admin@example.com

说明: 相当于PHP中的服务器参数: $_SERVER[“SERVER_ADMIN”]

SERVER_NAME

含义：当前运行脚本所在服务器的主机名，配置文件中 SERVER{ } 内 server_name 的值。

样例参考值: 如请求为 www.test.com/index.php，SERVER_NAME 值为 www.test.com

说明: 相当于PHP中的服务器参数: $_SERVER[“SERVER_NAME”]

SERVER_ADDR

含义：当前运行脚本所在服务器的 IP 地址

样例参考值: 127.0.0.1 或者 ::1

说明: 相当于PHP中的服务器参数: $_SERVER[“SERVER_ADDR”]

SERVER_PORT

含义：当前 WEB 服务器使用的端口，默认为80

样例参考值: 80

说明: 相当于PHP中的服务器参数: $_SERVER[“SERVER_PORT”]

SERVER_PROTOCOL

含义：请求页面时通信协议的名称和版本

样例参考值: HTTP/1.1

说明: 相当于PHP中的服务器参数: $_SERVER[“SERVER_PROTOCOL”]

SERVER_SOFTWARE

含义：服务器名称和版本

样例参考值: Apache/2.2.11 (Win32) PHP/5.2.9-1

说明: 相当于PHP中的服务器参数: $_SERVER[“SERVER_SOFTWARE”]

connection & request 部分参数

REMOTE_ADDR

含义：发起请求的客户端的IP地址

样例参考值: 127.0.0.1 正在浏览当前页面用户的 IP 地址。

说明: 相当于PHP中的服务器参数: $_SERVER[“REMOTE_ADDR”]

REMOTE_HOST

含义：浏览当前页面的用户的主机名。

样例参考值: 127.0.0.1

说明: 相当于PHP中的服务器参数: $_SERVER[“REMOTE_HOST”]

REMOTE_ADDR 和 REMOTE_HOST 都可能是路由上最后一个 proxy，而不是真实的客户端

REMOTE_PORT

含义：用户连接到服务器时所使用的端口

样例参考值: 2574 (变化的值)

说明: 相当于PHP中的服务器参数: $_SERVER[“REMOTE_PORT”]

REMOTE_USER

样例参考值: 发起请求的验证的用户名

说明: 相当于PHP中的服务器参数: $_SERVER[“REMOTE_USER”]

REMOTE_IDENT

含义: 发起请求的用户名（支持 IDENT 协议时有用）

REQUEST_METHOD

含义：访问页面使用的请求方法

样例参考值: GET、POST、HEAD、PUT

说明: 相当于PHP中的服务器参数: $_SERVER[“REQUEST_METHOD”]

SCRIPT_FILENAME

含义：当前执行脚本的绝对路径

样例参考值: C:/web/www/index.html

说明: 相当于PHP中的服务器参数: $_SERVER[“SCRIPT_FILENAME”]

PATH_INFO

含义：请求中真实脚本名称之后，查询语句（query string）之前的路径信息

样例参考值: 例如当前 URL 为 http://www.example.com/php/user.php/some/stuff?foo=bar，那么 PATH_INFO 将包含 /some/stuff。

说明: 相当于PHP中的服务器参数: $_REQUEST[“PATH_INFO”]

QUERY_STRING

含义：查询字符串

样例参考值: a=b&c=d&e=f

说明: 相当于PHP中的服务器参数: $_SERVER[“QUERY_STRING”]

AUTH_TYPE

含义：使用 HTTP 认证功能的类型

说明: 相当于PHP中的服务器参数: $_SERVER[“AUTH_TYPE”]

date and time 部分参数

TIME_YEAR

样例参考值: 2009

说明: 服务器获取当前的年份值

TIME_MON

样例参考值: 04

说明: 服务器获取当前的月份值

TIME_DAY

样例参考值: 22

说明: 服务器获取当前的日值

TIME_HOUR

样例参考值: 16

说明: 服务器获取当前时间的小时

TIME_MIN

样例参考值: 26

说明: 服务器获取当前时间的分钟

TIME_SEC

样例参考值: 34

说明: 服务器获取当前时间的秒

TIME_WDAY

样例参考值: 3

说明: 服务器获取当天是星期几, 从星期日-星期六, 数字从 0-6

TIME

样例参考值: 20090422162634

说明: 服务器获取当前的时间, 格式为: 年月日时分秒

specials 部分参数

API_VERSION

样例参考值: 20051115:21

说明: apache 的 API 版本信息.

THE_REQUEST

样例参考值: GET /share77.html HTTP/1.1

说明: 浏览器发给服务器的请求值. 不包括其他的头信息.

REQUEST_URI

样例参考值: /share77.html

说明: 浏览器请求的资源信息.

REQUEST_FILENAME

样例参考值: C:/webRoot/t/share77.html

说明: 被请求的资源的在磁盘的物理地址.

IS_SUBREQ

样例参考值: false

说明: 如果是 sub-request 则显示为 true, 否则为 false.

HTTPS

样例参考值: off

说明: 如果连接使用 SSL/TLS 模式, 则值为on , 否则值为off, 这个参数比较安全, 即使未载入 mod_ssl 模块
