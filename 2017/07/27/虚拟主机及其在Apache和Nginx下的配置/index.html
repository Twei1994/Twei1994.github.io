什么是虚拟主机

输入域名，打开网页，从网络协议方面看，就是访问某个 IP 的主机上的某个端口（默认是 80）。从服务器方面看来，就是监听到这个端口的访问时，把某个页面返回给客户端。

在配置好 Web 服务器后，默认有一个站点，但是我们会有这样的需求：只有一个服务器，但是搭建了多个站点，怎么才能实通过访问不同的域名或者端口实现对不同网站的访问？（具体到服务器里就是不同目录），这个时候就需要设置虚拟主机（VirtualHost）。

虚拟主机把一台服务器主机分成多个“虚拟”的主机，每台虚拟主机都可以具有独立的域名，具有完整的 Intemet 服务器功能（WWW、FTP、Email等），同一台主机上的虚拟主机之间是完全独立的。从网站访问者来看，每一台虚拟主机和一台独立的主机完全一样。

利用虚拟主机，不必为每个要运行的网站提供一台单独的 Nginx/Apache 服务器或单独运行一组 Nginx/Apache 进程。虚拟主机提供了在同一台服务器、同一组 Nginx/Apache 进程上运行多个网站的功能，并且网站之间相互独立各不干扰。

可以这样理解，服务器相当于是一个房子，里面放着站点的内容。这时又想提供一个站点，但是再买一个房子可能不是最好的选择，因为现在的房子还有大部分空着（CPU，内存，带宽）呢。这时最好的选择是将房子分成多个房间，每个房间放一个站点的内容。这就是虚拟主机。

配置虚拟主机的方式通常有 3 种：基于域名（最常见），基于端口和基于 IP。

Apache下的配置

在 Apache 的配置文件 httpd.conf 中去掉 Virtual hosts 配置项前的 #，引入 httpd-vhosts.conf 文件：

# Virtual hosts
Include conf/extra/httpd-vhosts.conf

以下的配置都在 conf/extra/httpd-vhosts.conf 中完成。

注：

  • 修改完配置文件后要重启 Apache 服务才能生效；
  • 可以通过 httpd -S 来查看已配置的虚拟主机；

基于域名

基于域名就是输入不同的域名，访问的是同一服务器上的不同目录。

因为我们并没有真实注册域名并绑定到本机 IP，我们通过修个 hosts 文件来实现，在 hosts 文件中加入下面这几行：

# Virtual Host Test
127.0.0.1 domain1.twei.site
127.0.0.1 domain2.twei.site

配置，在 httpd-vhosts.conf 中加入以下配置：

<VirtualHost *:80>
    ServerAdmin tjutwei@163.com
    ServerName domain1.twei.site
    DocumentRoot "D:/web/www/VirtualHostTest/domain1"
    # 访问权限，可以自己配置
    <Directory "/www/VirtualHostTest/domain1">
　　　　Options Indexes FollowSymLinks
　　　　AllowOverride None
　　　　Order allow,deny
　　　　Allow from all
　　</Directory>
    # 日志配置
    # ErrorLog logs/ error_log
    # TransferLog logs/ access_log
</VirtualHost>

<VirtualHost *:80>
    ServerAdmin tjutwei@163.com
    DocumentRoot "D:/web/www/VirtualHostTest/domain2"
    ServerName domain2.twei.site
</VirtualHost>

上面的配置实现了输入域名 domain1.twei.site 时访问 D:/web/www/VirtualHostTest/domai1，输入域名 domain2.twei.site 时访问 D:/web/www/VirtualHostTest/domai2。

在使用虚拟主机后，默认的站点为第一个虚拟主机，如果输入不是特定的域名，而是其它的例如 localhost时，访问的是 D:/web/www/VirtualHostTest/domai1 这个目录而不是之前配置的 DocumentRoot。

注：在 Apache2.4 版本之后，NameVirtualHost 配置项已失效，如果在 httpd-vhosts.conf 中使用了该配置，在重启 Apache 服务时会提示如下错误：

NameVirtualHost has no effect and will be removed in the next release /usr/local/apache/conf/extra/httpd-vhosts.conf:23

基于端口

基于端口就是输入不同的端口，访问的是同一服务器上的不同目录。

对 httpd.conf 中增加监听端口：

Listen 80
Listen 81

在 httpd-vhost.conf 中添加如下配置：

<VirtualHost *:80>
    DocumentRoot "D:/web/www/VirtualHostTest/domain1"
</VirtualHost>

<VirtualHost *:81>
    DocumentRoot "D:/web/www/VirtualHostTest/domain2"
</VirtualHost>

注：不要添加 ServerName 字段；<Directory> 和日志配置可以选填；

上面的配置实现了输入域名 localhost:80 时访问 D:/web/www/VirtualHostTest/domai1，输入域名 localhost:81 时访问 D:/web/www/VirtualHostTest/domai2。

域名和端口的混合使用

也可以混合使用域名和端口，实现类似下面的功能：

domain1.twei.stie:80  =====>  /www/VirtualHostTest/domai1
domain1.twei.stie:81  =====>  /www/VirtualHostTest/domai2

相关的配置如下：

httpd.conf

Listen 80
Listen 81

httpd-vhost.conf

<VirtualHost *:80>
    ServerName domain1.twei.site
    DocumentRoot "D:/web/www/VirtualHostTest/domain1"
</VirtualHost>

<VirtualHost *:81>
    ServerName domain2.twei.site
    DocumentRoot "D:/web/www/VirtualHostTest/domain2"
</VirtualHost>

实现的效果如下：

domain1.twei.stie:80  =====>  /www/VirtualHostTest/domai1
domain1.twei.stie:80  =====>  /www/VirtualHostTest/domai1
domain1.twei.stie:81  =====>  /www/VirtualHostTest/domai2
domain1.twei.stie:81  =====>  /www/VirtualHostTest/domai2

通过 httpd -S 查看：

*:80                   domain1.twei.site
*:81                   domain2.twei.site

80 端口的默认站点为 domain1.twei.site 对应的站点，81 端口的默认站点为 domain2.twei.site 对应的站点。

基于IP

将不同的网站挂在不同的 IP 上，访问不同的 IP，所看到的是不同网站。

因为一般服务器没那么多公网 IP，而且用户一般都是用域名访问的。所以这个不常用，但是可以用来测试和学习。

配置就是将 <VirtualHost> 中添加上 IP 地址：

<VirtualHost 192.168.0.111>
    DocumentRoot "D:/web/www/VirtualHostTest/domain1"
</VirtualHost>

<VirtualHost 192.168.0.112>
    DocumentRoot "D:/web/www/VirtualHostTest/domain2"
</VirtualHost>

注：如果在重启服务时出现下面的错误，加上 ServerName，任意名称均可。

Failed to resolve server name for 192.168.1.115 (check DNS) -- or specify an explicit ServerName

Nginx下的配置

Nginx 下一个 server 就是一个虚拟主机，所以要配置虚拟主机只要增加 http 下的 server 即可。

基于域名

首先还是修改 hosts 文件，Ubuntu 的 hosts 文件位于 /etc/hosts，在末尾添加：

# Virtual Host Test
127.0.0.1 domain1.twei.site
127.0.0.1 domain2.twei.site

然后修改 nginx.conf，增加以下配置：

server {  
       listen       80;  
       server_name  domain1.twei.site;  
       #charset koi8-r;  
       #access_log  logs/host.access.log  main;  
       location / {  
           root   html/domain1;  
           index  index.html index.htm;  
       }  
}   
server {  
       listen       80;  
       server_name  domain2.twei.site;  
       #charset koi8-r;  
       #access_log  logs/host.access.log  main;  
       location / {  
           root   html/domain2;  
           index  index.html index.htm;  
       }  
}

然后通过 /usr/local/nginx/sbin/nginx -s reload 重启 Nginx 服务。

通过浏览器可以验证，域名 domain1.twei.site 的根目录为 /usr/local/nginx/html/domain1，而域名 domain2.twei.site 的根目录为 /usr/local/nginx/html/domain2。

注：与 Apache 不同的是，现在 localhost 仍然对应 /usr/local/nginx/html/
这个路径。原因是我们没有修改之前的 server 配置。

基于端口

修改 nginx.conf，增加如下配置：

server {  
        listen       80;  
        server_name  localhost;  
        #charset koi8-r;  
        #access_log  logs/host.access.log  main;  
        location / {  
            root   html/domain1;  
            index  index.html index.htm;  
        }         
}  
server {  
        listen       81;  
        server_name  localhost;  
        #charset koi8-r;  
        #access_log  logs/host.access.log  main;  
        location / {  
            root   html/domain2;  
            index  index.html index.htm;  
        }          
}

上面的配置实现了通过 80 端口访问的是 /usr/local/nginx/html/domain1，通过 81 端口访问的是 /usr/local/nginx/html/domain2。

当然，Nginx 也可以混合使用基于域名和端口的虚拟主机配置。

基于IP

首先，需要我们给本机增加一个虚拟网卡，从而让本机有多个 IP。

通过 ifconfig 可以查看本机的 IP，目前我的电脑 IP 是 192.168.1.116，然后通过 ifconfig eth0:0 192.168.1.117 netmask 255.255.240.0 up 就给本机增加了一个名称为 eth0:0 的虚拟网卡，它的 IP 为 192.168.1.117。

接下来配置 nginx.conf，增加另外两个 server 节点：

server {  
        listen       80;  
        server_name  192.168.1.116;  
        #charset koi8-r;  
        #access_log  logs/host.access.log  main;  
        location / {  
            root   html/domain1;  
            index  index.html index.htm;  
        }         
    }  

server {  
        listen       80;  
        server_name  192.168.1.117;  
        #charset koi8-r;  
        #access_log  logs/host.access.log  main;  
        location / {  
            root   html/domain2;  
            index  index.html index.htm;  
        }          
    }

这就是实现了在同一台硬件服务器上，虚拟出两个 IP 地址，使用 1 个 nginx 服务器，分别对两个 IP 进行访问，分别跳转到不同的 html 页面。
