TCP常见问题

  • TCP连接的建立和断开
  • TCP报文格式

常见问题

TCP连接有哪些特点？

最大的特点是 TCP 是可靠的，也就是说通过 TCP 连接传送的数据，无差错，不丢失，不重复，且按序到达。当然，这里的无差错并不是指 TCP 连接一定是安全的。

其他的特点包括：面向连接，面向字节流，有流量控制和拥塞控制，点对点服务等。

TCP连接是如何保证可靠性的？

我的理解是通过以下几点：

  1. 通过数据流的方式传输，将数据截成合理的长度；
  2. 对每段报文进行编号，保证数据流的顺序；
  3. 收到数据进行重排序，并且丢弃重复数据；
  4. 收到报文后，给出确认响应；
  5. 超时重发；
  6. 校验和；
  7. 流量控制（针对本程序的接受能力）和拥塞控制（针对整个网络）。

TCP和UDP的区别？

他们的区别主要有以下几点：

  1. TCP 的面向连接的，在发送数据之前要建立连接；UDP 是无连接的，即发送数据之前不需要建立连接；
  2. TCP 提供可靠的服务，通过 TCP 连接传送的数据，不丢失，不重复，且按序到达。UDP 尽最大努力交付，不保证可靠交付；
  3. TCP 面向字节流，把数据看成一连串无结构的字节流；UDP 是面向报文的，对应用程序的报文既不拆分也不合并；
  4. TCP 有流量控制和拥塞控制，UDP 没有拥塞控制；
  5. TCP 连接只能是点到点的，UDP 支持一对一，一对多，多对一和多对多的交互通信；
  6. TCP首部至少开销 20 字节，UDP 的首部结构简单，开销小，只有8个字节；
  7. 由于 TCP 要维护连接和保证可靠性，所以对系统资源的要求较多， UDP 相对要小得多。
