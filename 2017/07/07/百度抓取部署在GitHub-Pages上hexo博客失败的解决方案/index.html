百度抓取部署在GitHub Pages上hexo博客失败的解决方案

前言

前两天为博客增加访问人数和博客阅读数统计，并做了一些 SEO 优化，希望能增加人气。其中有一项是向百度站长提交自己网站的 sitemap。

但是今天查了下，发现并没有被抓取，测试了下，报的是 403 Forbidden 错误。

HTTP/1.0 403 Forbidden
Cache-Control: no-cache
Connection: close
Content-Type: text/html

原因是 GitHub Pages 屏蔽了百度爬虫的爬取。

Sorry for the trouble with this. We are currently blocking the Baidu user agent from crawling GitHub Pages sites in response to this user agent being responsible for an excessive amount of requests, which was causing availability issues for other GitHub customers.

This is unlikely to change any time soon, so if you need the Baidu user agent to be able to crawl your site you will need to host it elsewhere.

Apologies again for the inconvenience.

解决方案

除了采用 sitemap 自动提交之外，百度还提供了另外3种方式。

如何选择链接提交方式
1、主动推送：最为快速的提交方式，推荐您将站点当天新产出链接立即通过此方式推送给百度，以保证新链接可以及时被百度收录。
2、自动推送：最为便捷的提交方式，请将自动推送的JS代码部署在站点的每一个页面源代码中，部署代码的页面在每次被浏览时，链接会被自动推送给百度。可以与主动推送配合使用。
3、sitemap：您可以定期将网站链接放到sitemap中，然后将sitemap提交给百度。百度会周期性的抓取检查您提交的sitemap，对其中的链接进行处理，但收录速度慢于主动推送。
4、手动提交：一次性提交链接给百度，可以使用此种方式。

其中，手动提交要自己输入所有要提交的链接，太麻烦，不予考虑。

自动推送只要添加以下JS代码到博客的配置页（footer.swig）就可以。

<script>
(function(){
    var bp = document.createElement('script');
    var curProtocol = window.location.protocol.split(':')[0];
    if (curProtocol === 'https') {
        bp.src = 'https://zz.bdstatic.com/linksubmit/push.js';        
    }
    else {
        bp.src = 'http://push.zhanzhang.baidu.com/push.js';
    }
    var s = document.getElementsByTagName("script")[0];
    s.parentNode.insertBefore(bp, s);
})();
</script>

我采用的是主动推送的方式：

写了一个简单的PHP脚本来实现，每次 hexo g 之后，更新了 baidusitemap.xml，就可以使用这个脚本主动提交链接给百度爬虫（当然最后要 hexo d 部署到 GitHub 上去）。

<?php
// 解析xml文件，生成链接数组urls
$file = "./public/baidusitemap.xml";
$xml_obj = simplexml_load_file($file);
$urls = array();
foreach ($xml_obj as $key => $article) {
    $article_url = array();
    $article_url = (array)$article->loc;
    $urls[] = current($article_url);
}

// 调用接口提交网站链接
$api = 'http://data.zz.baidu.com/urls?site=www.twei.site&token=4fYKrhRo0p8mDk7N';
$ch = curl_init();
$options =  array(
    CURLOPT_URL => $api,
    CURLOPT_POST => true,
    CURLOPT_RETURNTRANSFER => true,
    CURLOPT_POSTFIELDS => implode("\n", $urls),
    CURLOPT_HTTPHEADER => array('Content-Type: text/plain'),
);
curl_setopt_array($ch, $options);
$result = curl_exec($ch);

// 结果写入日志中
$num = count($urls);
$time = date("Y-m-d H:i:s");
$content = $time. " @ ". $num. " @ ". $result. PHP_EOL;

$file = "upload_sitemap.log";
$fp = fopen($file, "a+") or die("fail to open log file");
fwrite($fp, $content);
fclose($fp);
?>

使用方法：

  • 要求本机有PHP环境，如果没有，先装好PHP
  • 将脚本保存在网站根目录下，保存成 upload_sitemap_to_baidu.php
  • 每次写完新的博文，hexo g 和 hexo d 之后，继续执行 php -f upload_sitemap_to_baidu.php
  • 执行结束后，可以通过根目录下生成的 upload_sitemap.log 文件查看每次提交的结果
