Apache实现伪静态–Rewrite模块学习

正文

什么是伪静态？

要说伪静态，首先要说静态和动态文件。

静态文件就是哪些不会改变的文件，如 html文件，图片，视频等。这些文件内容都是固定的。而动态文件正好相反，是指哪些内容不固定的文件，如 php 文件就是动态文件。它通过参数来从数据库中取数据放在页面中，不同的参数返回不同的内容。

伪静态介于动态和静态之间，它是将动态的页面用静态的形式展示的技术，例如用 user1.html 来代替 user.php?id=1，从外部来看，就像是一个静态的页面一样，类似的映射过程就实现了伪静态。

为什么要用伪静态？

一句话：为了搜索引擎友好。

与静态网页不同，动态网页往往要输入参数，搜索引擎比较烦这种带问号的动态网页，因为参数可以随便加，而返回内容基本却不变，所以会对这种网页降权。当用了伪静态时，动态网页看起来就像是静态的一样，就会被搜索引擎收录。

除了搜索引擎友好，还有以下优点：真正的静态页面空间储存量大，进行删除或者更新这些 html 文件时可造成大量文件碎片，破坏磁盘坏道，而伪静态可以缓解磁盘的压力；

当然，它也是有缺点的：

  1. 增加了 CPU 的负担，由于伪静态需要判断当前访问地址需不需要解析成动态页面，所以 CPU 占有量会上升；
  2. 没有纯静态页面快；

Apache配置

加载rewrite模块

修改 Apache 配置文件 httpd.conf 中找到 LoadModule rewrite_module modules/mod_rewrite.so 这句，去掉前边的注释符号 “#” ，如果没有则添加这句。

为了允许在任何目录中使用 “.htaccess” 文件，将 Directory 中的 “AllowOverride” 改成 “All” （默认为 “None” ）：

# AllowOverride controls what directives may be placed in .htaccess files.
# It can be “All”, “None”, or any combination of the keywords:
# Options FileInfo AuthConfig Limit
#
AllowOverride All

在 Windows 系统下不能直接建立 “.htaccess” 文件，可以在命令行下使用 echo a> .htacces 建立，然后使用记事本编辑。

实例

编辑 .htaccess 文件，添加下面的代码，实现 user/1 ===> user.php?id=1 的映射：

# 开启rewrite功能
RewriteEngine on
# 匹配规则
RewriteRule ^user/([0-9]+)$ user.php?id=$1

重定向配置详解

其实，.htaccess 作为 Apache 的分布式配置文件，它的功能不止重定向一个，还可以实现文件夹密码保护、自定义错误页面、改变你的文件扩展名、封禁特定 IP 地址的用户、只允许特定 IP 地址的用户、禁止目录列表，以及使用其他文件作为 index 文件等一些功能。

这里就只列出用 .htaccess 实现重定向时用的一些语法：

  • RewriteEngine

    RewriteEngine On|Off

通过配置为 On 或者 Off 来开启或停用 rewrite 功能，它不会自动继承，因此每个想用 rewrite 功能的虚拟主机目录都要加上这个指令。

  • RewriteBase

    RewriteBase URL-path

用于设定重写的基准 URL。默认值是 RewriteBase physical-directory-path。

  • RewriteCond

    RewriteCond TestString CondPattern [flags]

定义了一个规则的条件，即在 TestString 满足 CondPattern 时，执行下面的 RewriteRule 指令。可以有一个或多个RewriteCond指令。

类似程序中的 if 语句一样，表示如果符合某个或某几个条件则执行 RewriteCond 下面紧邻的 RewriteRule 语句。

  • RewriteRule

    RewriteRule Pattern Substitution [flags]

RewriteRule 指令是重写引擎的根本，每个指令定义一个简单的重写规则，就是用 Substitution 去替换 Pattern。

定义的顺序尤为重要, 因为在执行时，就是按照定义的顺序逐一生效的。

RewriteCond RewriteRule 实例

RewriteEngine on
RewriteCond  %{HTTP_USER_AGENT}  ^Mozilla//5/.0.*
RewriteRule  index.php            index.m.php
RewriteCond  %{HTTP_USER_AGENT}  ^Lynx.*
RewriteRule  index.php            index.L.php 
RewriteRule  index.php            index.b.php

上面语句的作用是当你是用火狐浏览器访问 index.php 这个文件的时候，会自动让你访问到 index.m.php 这个文件，当你用一些移动终端访问的时候，会让你对 index.php 这个文件的访问实际访问的是 index.L.php，如果你是用其它的浏览器访问的时候，会让你跳到 index.b.php。

在说形象一点，上面的语句就等同于程序里面的下面语句（以 PHP 语句为例）:

if($_SERVER['HTTP_USER_AGENT'] == 'Mozilla/5.0')
{
//跳转到对index.m.php的访问
}
else if($_SERVER['HTTP_USER_AGENT'] == 'Lynx')
{
//跳转到对index.L.php的访问
}
else
//跳转到对index.b.php的访问
