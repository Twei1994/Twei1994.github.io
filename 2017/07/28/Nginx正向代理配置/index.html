什么是正向代理

正向代理，也就是我们常说的代理，是服务于客户端的，作用就是转发我们的 WEB 请求并返回结果，可以看作是客户端和网站服务器中间的转发器。

通常在两种情况下我们会使用代理：

  • 当前客户端无法访问某些地址

例如在局域网中，只有 1 台机器能联网，其他机器都不能；或者说 GFW；这种时候我们可以先联上一台能访问目标网站的服务器，然后通过它去访问目标网站，也就是所谓的“翻墙”。

  • 希望隐藏客户端的信息

当我们使用代理访问 Internet 时，从外部看，只能获得代理服务器的信息，不能得到客户端的信息。

我突然想起高中时的一件事：高中在外地上的，所以是住校。住校平时是不让出去买东西的。但是我又喜欢看杂志。所以只能拜托跑校的同学帮我买。这个跑校的同学就是我的正向代理。

Nginx正向代理配置

nginx.conf 文件：

server {
    # 配置DNS解析IP地址，比如 Google Public DNS，以及超时时间（5秒）
    resolver 8.8.8.8;    # 必需
    resolver_timeout 5s;

    # 监听端口
    listen 8080;

    access_log  /home/reistlin/logs/proxy.access.log;
    error_log   /home/reistlin/logs/proxy.error.log;

    location / {
        # 配置正向代理参数
        proxy_pass $scheme://$host$request_uri;
        # 解决如果URL中带"."后Nginx 503错误
        proxy_set_header Host $http_host;

        # 配置缓存大小
        proxy_buffers 256 4k;
        # 关闭磁盘缓存读写减少I/O
        proxy_max_temp_file_size 0;
         # 代理连接超时时间
        proxy_connect_timeout 30;

        # 配置代理服务器HTTP状态缓存时间
        proxy_cache_valid 200 302 10m;
        proxy_cache_valid 301 1h;
        proxy_cache_valid any 1m;
    }
}

注：

  • 配置 nginx 正向代理服务，一般是配置到一个 server 块中。注意，在该 server 块中，不要出现 server_name 指令，即不要设置虚拟主机的名称和 IP；
  • resolver 是必需的，如果没有该指令， nginx 无法处理接收到的域名；
  • nginx 代理服务不支持正向代理 HTTPS 站点；

使用代理服务器

配置完成后，重启 nginx 就可以生效了。假设我们配置正向代理的服务器 IP 为 192.168.1.9。

Linux

在需要访问外网的机器上执行以下命令即可：export http_proxy=http://192.168.1.9：8080
