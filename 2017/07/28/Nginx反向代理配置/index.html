什么是反向代理

反向代理，是给服务器端提供方便的，当客户端向站点提出请求时，会请求代理服务器。然后，代理服务器将客户端的请求发送到真正的内容服务器或者 WEB 服务器。反向代理也就是服务器的正向代理，只不过站在客户端的角度来看，就是反向代理了。

使用反向代理主要是出于这两个原因：

  • 保护真实服务器

在真实的服务器上，可能有某些必须保持安全的敏感信息，如信用卡号数据库等，我们不希望被客户端访问到。我们可以在防火墙外部设置一个代理服务器作为真实服务器的替身。

当外部客户机尝试访问内容服务器时，会将其送到代理服务器。代理服务器通过防火墙中的特定通路，将客户机的请求发送到内容服务器。这样涉及到安全方面的请求就可以被过滤掉。

  • 用做服务器集群的负载均衡器

如果 WEB 服务器每天都会接收大量的请求，则可以使用代理服务器来担当负载均衡器，将 WEB 请求分配给不同的服务器，降低多个请求访问一个服务器从而导致服务器崩溃的危险。

Nginx反向代理配置

server {  
    listen       80;  
    server_name  10.0.0.136;  

    location / {
        # 被代理的服务器IP，就是真实服务器的IP地址  
        proxy_pass http://10.0.0.137;    # 必需

        # 以下是一些反向代理的配置（非必需）
        proxy_redirect             off;

        # 后端的Web服务器可以通过 X-Forwarded-For 获取用户真实IP
        proxy_set_header           Host $host; 
        proxy_set_header           X-Real-IP $remote_addr; 
        proxy_set_header           X-Forwarded-For $proxy_add_x_forwarded_for;

         # 允许客户端请求的最大单文件字节数
        client_max_body_size       10m; 

        # 缓冲区代理缓冲用户端请求的最大字节数
        client_body_buffer_size    128k;

         # nginx跟后端服务器连接超时时间(代理连接超时)
        proxy_connect_timeout      300;

         # 后端服务器数据回传时间(代理发送超时)
        proxy_send_timeout         300;

        # 连接成功后，后端服务器响应时间(代理接收超时)
        proxy_read_timeout         300; 

        # 设置代理服务器（nginx）保存用户头信息的缓冲区大小
        proxy_buffer_size          4k;

        #proxy_buffers缓冲区，网页平均在32k以下的话，这样设置
        proxy_buffers              4 32k;

         #高负荷下缓冲大小（proxy_buffers*2）
        proxy_busy_buffers_size    64k;

        #设定缓存文件夹大小，大于这个值，将从upstream服务器传  
        proxy_temp_file_write_size 64k; 
    }  
}

上面的配置实现了将 10.0.0.136 服务器作为 10.0.0.137 的反向代理。

暴露给客户端的 IP 地址为 10.0.0.136。当有客户访问时，Nginx 就把请求转给 10.0.0.137。客户端获取的内容也是存放在 10.0.0.137 上。
