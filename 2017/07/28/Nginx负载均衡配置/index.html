什么是负载均衡

虽然 Nginx 的并发性能很好，单台最多能支持 50000 的并发，但是随着用户的增多和业务的变化，并发量会越来越高。这时一台服务器就无法满足业务需求了，需要使用多台服务器。将客户端的请求转发到多台服务器的过程就是负载均衡。

通过负载均衡也可以避免单台服务器挂掉影响整个系统。

Nginx配置

upstream模块

与之前的博客介绍过的用 Nginx 做反向代理时的配置相比，负载均衡要增加一个 upstream 模块。

upstream 是 Nginx 的 HTTP Upstream 模块，这个模块通过一个简单的调度算法来实现客户端 IP 到后端服务器的负载均衡。使用时，先通过 upstream 指令指定一个负载均衡器的名称（可以任意指定），在后面需要用到的地方（proxy_pass）直接调用即可。

调度算法

Nginx 的负载均衡模块目前支持 4 种调度算法，下面进行分别介绍，其中后两项属于第三方调度算法：

  • 轮询（默认）。每个请求按时间顺序逐一分配到不同的后端服务器，如果后端某台服务器宕机，故障系统被自动剔除，使用户访问不受影响。通过 Weight 指定轮询权值，Weight 值越大，分配到的访问机率越高，主要用于后端每个服务器性能不均的情况下；
  • ip_hash。 每个请求按访问 IP 的 hash 结果分配，这样来自同一个 IP 的访客固定访问一个后端服务器，有效解决了动态网页存在的 session 共享问题；
  • fair。 这是比上面两个更加智能的负载均衡算法。此种算法可以依据页面大小和加载时间长短智能地进行负载均衡，也就是根据后端服务器的响应时间来分配请求，响应时间短的优先分配。 Nginx 本身是不支持 fair 的，如果需要使用这种调度算法，必须下载 Nginx 的 upstream_fair 模块；
  • url_hash。 此方法按访问 url 的 hash 结果来分配请求，使每个 url 定向到同一个后端服务器，可以进一步提高后端缓存服务器的效率。 Nginx 本身是不支持 url_hash 的，如果需要使用这种调度算法，必须安装 Nginx 的 hash 软件包。

upstream支持的状态参数

在 Upstream 模块中，在指定后端服务器的 IP 地址和端口的同时，还可以设定每个后端服务器在负载均衡调度中的状态。常用的状态有：

  • weight，使用轮询调度策略时，服务器的权重，权重大的被选中概率高；
  • down，表示当前的 server 暂时不参与负载均衡；
  • backup，预留的备份机器。当其他所有的非 backup 机器出现故障或者忙的时候，才会请求 backup 机器，因此这台机器的压力最轻；
  • max_fails，允许请求失败的次数，默认为 1。当超过最大次数时，返回proxy_next_upstream 模块定义的错误；
  • fail_timeout，在经历了 max_fails 次失败后，暂停服务的时间。 max_fails 可以和 fail_timeout 一起使用。

注：当负载调度算法为 ip_hash 时，后端服务器在负载均衡调度中的状态不能是 weight 和 backup。

配置

下面的例子实现了用 Nginx 做负载均衡器实现将客户端请求分配到两位两台 Apache 服务器上。它们的 IP 地址如下：

nginx:   192.168.4.72   
apache:  192.168.4.69   
apache:  192.168.4.68

修改 Nginx 的配置文件 nginx.conf：

http {

    ....

    upstream proxy_test {
           server 192.168.4.69:80 weight=1;
           server 192.168.4.68:80 weight=1;
           #ip_hash; 
     }

    server {
        listen       80;
        server_name  localhost;
        charset utf8;

        location / {
            # 这里的proxy_test是上面的负载的名称，可以是ip加端口或url 
            proxy_pass       http://proxy_test; 

            proxy_set_header Host      $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        }
    }
}

注：upstream 是定义在 server{ } 之外的，不能定义在 server{ } 内部。

上面的配置就实现负载均衡，当访问 Nginx 时，得到的返回是两个 Apache 服务器上的内容。
