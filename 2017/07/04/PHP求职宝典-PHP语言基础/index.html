优化以下代码

<?
echo ("<p>Search results for query: " . $_GET['query'] . ".</p>");
?>

分析：首先，它使用了短标签 <? ?>，不符合 XML 规范，所以不建议使用；其次，最大的问题是将用户的输入直接输出，产生了跨站脚本攻击（XSS）的漏洞，应该使用 htmlspecialchars()函数屏蔽非法字符；最后还有一些小的地方可以改进。

结果：

<?php
if (isset($_GET['query'])) {
    echo '<p>Search results for query: ' , htmlspecialchars($_GET['query'], ENT_QUOTES) , '.</p>';
}
?>

修改部分：

  • 将PHP短标签换成 <?php ?>，更符合XML规范；
  • 在输出 $_GET['query'] 前，判断它是否存在、被定义；
  • 去掉 echo 中多余的括号；
  • 用单引号代替双引号，用逗号代替句号，提高 echo 的效率；
  • 在 htmlspecialchars 函数中使用参数 ENT_QUOTES 来确保引号会被转义，增加安全性。

PHP中数据类型的判断和转换方法

PHP的数据类型有7种，分别是 boolean（或为bool）、integer（或为int）、float、string、array、object 和 null。

判断数据类型的方法是用：is_bool()、is_int()、 is_float()、is_string()、is_array()、is_object()。另外还有一个函数 is_numeric() 用于判断变量是否为数字或者数字组成的字符串。

数据类型的转换方式：

  • 在变量前加上用括号括起来的数据类型，如 (int) (bool) (float) (string) (array) (object)；
  • 使用3个具体类型的转换函数，intval()、floatval()、strval()；
  • 使用函数 settype(var, type)；

PHP何时区分大小写

区分大小写的时候：

  • 变量名
  • 常量名，通常写成大写

不区分大小写的时候：

  • 函数名、方法名、类名，在调用时尽量使用和定义时相同的名字；
  • 魔术常量不区分大小写，如 __LINE__，__FUNCTION__等；
  • NULL，TRUE，FLASE不区分；
  • 类型转换时，如(string) 和（STRING）是相同的；

count()函数的输出

count() 函数用于统计数组中元素的个数，对其他类型，输出结果为1。

echo count("a");
echo count(1);
echo count(2.4);
echo count(false);
echo count(new Node());

以上输出结果都为1。但 count(null) 输出为0。

php.ini中safe_mode选项有什么用？开启后会影响到哪些函数？

safe_mode 是指PHP的安全模式，开启之后，PHP将会检查当前脚本的拥有者是否和被操作文件的拥有者相同，目的是提供一个相对安全的共享环境。

因为Linux是多用户的，所以可能出现你的文件被别人写入或者读取的现象。打开PHP的安全模式后，一些与文件操作相关的函数将被限制，同样的一些php拓展中的函数也会受到影响。

开启后会受到影响的函数有：

chdir, move_uploaded_file, chgrp, parse_ini_file, chown, rmdir, copy, rename, fopen, require, highlight_file, show_source, include, symlink, link, touch, mkdir, unlink, basename, pathinfo

以下执行命令的函数也会受到影响：

exec, shell_exec, passthru, system, popen

PHP中的静态变量

指出下面代码的输出结果：

<?php 
$count = 5;
function get_count() {
    static $count = 0;
    return $count++;
}
echo $count;
++$count;
echo get_count();
echo get_count();
?>

输出结果是： 5 0 1

静态变量类型说明符是static。

静态变量属于静态存储方式，其存储空间为内存中的静态数据区，该区域中的数据在整个程序的运行期间一直占用这些存储空间（在程序整个运行期间都不释放），也可以认为是其内存地址不变，直到整个程序运行结束。

静态变量虽在程序的整个执行过程中始终存在，但是在它作用域之外不能使用。

所以在自定义函数内的静态变量 count 的值在第二次调用时仍会保留。

PHP中的全局变量

指出下面代码的输出结果：

<?php
$GLOBALS['a'] = "hello";
$b = 2017;

function get_value() {
    global $b;
    $a = 0;
    return $b++;
}

get_value();
echo $a;
echo $b;
?>

输出结果是: hello 2018

在一个函数中引入外部的一个变量，如果该变量没有通过参数传递进来，那么就通过 global 引入进来或者通过 $GLOBALS 数组访问。

mysql_fetch_row()和mysql_fetch_array()的区别

二者都是从数据库结果集中取一条记录。区别在于 mysql_fetch_row() 只会以数字数组的方式返回结果，mysql_fetch_array() 可以通过参数设置（MYSQL_ASSOC，MYSQL_NUM、MYSQL_BOTH）返回结果的方式，分别表示关联数组、数字数组和二者都取（默认）。

数字数组：Array([0] => Adams [1] => John [2] => London)

关联数组：Array([LastName] => Adams [FirstName] => John [City] => London)

二者都取：Array([0] => Adams [LastName] => Adams [1] => John [FirstName] => John [2] => London [City] => London)

如何实现字符串翻转？

两种方式，第一用自带的 strrev() 函数，第二自己写函数。

function strreverse($str) {
    $res = "";
    for ($i = strlen($str)-1; $i >= 0; $i--) {
        $res .= $str[$i];
    }
    return $res;
}

原来可以像取数组元素一样获取字符串中的字符。

PHP中的进制转换

首先，列出各个进制的英文

二进制： binary –> bin

八进制： octal –> oct

十进制： decimal –> dec

十六进制： hexadecimal –> hex

所以，对应的进制转换函数就是将原进制和目标进制合起来：例如十进制转二进制函数 decbin()，八进制转十进制函数 octdec()。

另外，进制转换推荐使用这个函数 base_convert()

base_convert – 在任意进制之间转换数字

string base_convert ( string number, int frombase, int tobase )

返回一字符串，包含 number 以 tobase 进制的表示。number 本身的进制由 frombase 指定。frombase 和 tobase 都只能在 2 和 36 之间（包括 2 和 36）。高于十进制的数字用字母 a-z 表示，例如 a 表示 10，b 表示 11 以及 z 表示 35。

对二维数组按照某一列进行排序

<?php
$list = array(
    array('num' => 7, 'value' => 4),
    array('num' => 3, 'value' => 35),
    array('num' => 38, 'value' => 27),
);
$res = array_sort($list, "value", "ASC");
print_r($res);

function array_sort(array $list, $col, $order) {
    $col_key = array();
    foreach ($list as $key => $value) {
        $col_key[$key] = $value[$col];
    }
    if ($order === "ASC") {
        asort($col_key);
    }else{
        arsort($col_key);
    }
    $res = array();
    foreach ($col_key as $key => $value) {
        $res[] = $list[$key];
    }
    return $res;
}

关键是要想到将要排序那列的值和 key 值进行对应，然后按照 key 进行排序。

禁用Cookie后session还能用吗？

答案是可以的。

默认情况下， PHP 的 session 是需要 Cookie 支持的。

session 的工作原理： session 是存放在服务器端的。当客户端发起请求并需要创建一个 session 时，服务器首先检查这个客户端的请求里是否已包含了 session id。如果包含则说明之前已经为此客户端创建过 session，服务器就按照 session id 把相关文件检索出来使用。如果检索不到或者客户端请求不包含 session id，则为此客户端新建一个 session 文件并且生成一个对应的 session id。这个session id将被在本次响应中返回给客户端保存。

保存这个session id的方式可以采用cookie，这样在交互过程中浏览器可以自动的按照规则把这个标识发挥给服务器。

但cookie可以被人为的禁止，则必须有其他机制以便在cookie被禁止时仍然能够把session id传递回服务器。 经常被使用的一种技术叫做URL重写，就是把session id直接附加在URL路径的后面。还有一种技术叫做表单隐藏字段。就是服务器会自动修改表单，添加一个隐藏字段，以便在表单提交时能够把session id传递回服务器。

写一个函数，实现遍历一个文件夹下的所有文件和子文件夹

function scandir($dir) {
    $files=array();
    if(is_dir($dir)) {
        if($handle = opendir($dir)) {
            while(($file = readdir($handle)) !== false) {
                if($file != "." && $file != "..") {
                    if(is_dir($dir."/".$file)) {   //是文件夹
                        $files[] = candir($dir."/".$file);
                    }else{        //文件
                        $files[]=$dir."/".$file;
                    }
                }
            }
            closedir($handle);
            return $files;
        }
    }
}
