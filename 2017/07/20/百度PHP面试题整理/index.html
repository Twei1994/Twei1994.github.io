大文件排序

从100万条字符串中找出热门的10条

这是一道大数据题，要采用 Hash 表 + Top K + 分治法来解决。

首先，因为要找的是最热门的，即出现次数最多的，此时还不是 Top K 问题，所以要先统计每个字符串出现的次数，这个工作利用 HashTable 实现。将字符串作为 key，利用一个哈希函数将它映射到某个地址，这个地址存储它出现的次数。这样经过一次变量后，我们得到了一个哈希表，key 为所有字符串，value 为它出现的次数。时间复杂度为 o(N)，N 为 100万。哈希表的大小为 N’，N’ 是不重复的字符串的个数。

那么，热门也就是出现次数最多的，此时问题转化成从 N’ 个数里面找出最大的 10 个数，也就是 Top K 问题，我们可以通过小顶堆来实现。也就是我们创建一个大小为 10 的小顶堆，每当有新的数过来时，和堆顶进行比较，如果大于堆顶，则替换堆顶元素并更新小顶堆（这个过程算法复杂度为 o(log K)）。这样一次结束后，就得到了最大的 10 个数。时间复杂度为 o(N’*log K)，K 在这里等于 10。

也可以在其中采用分治法，将 100 万条数据分成 1000 份（通过哈序函数取余实现）。对这 1000 份数据，利用哈希表统计出现次数并得出每份数据中最大的 10 个，然后在1000*10 这么多数中再找出 10 个最大的。

实现intval()函数

<?php
// 测试范例  
$tests = array(
        '123',    // 123
        '-123',   // -123
        'abc',    // 0
        '-abc',   // 0
        '123abc', // 123
        ' ',      // 0
        "",          // 0
        " 123",   // 123
    );
// 执行每个测试样例
array_walk($tests, 'MyIntval');

/**
 * [MyIntval 实现php字符串转数字函数intval()]
 * @param [string] $str
 */
function MyIntval($str) {
    $res = 0;
    $len = strlen(trim($str));
    if ($len > 0) {   // 排除null
        $sign = 1;  // 符号位
        $i = 0;
        if ($str[$i] == "+") {
            ++$i;
        }elseif ($str[$i] == "-") {
            $sign = -1;
            ++$i;
        }
        while ($i < $len) {
            if ($str[$i] < '0' || $str[$i] > '9') {   // 排除非数字的字符
                break;
            }
            $temp = $str[$i] - "0";  // 与"0"相减将字符转成数字
            $res = $res*10+$temp;
            ++$i;
        }
        $res *= $sign;
    }
    // 为了配合array_walk，正常应该用 return $res
    echo $res,'<br>';
}

Linux同时kill掉所有的Nginx进程

// 通过 killall 命令
killall nginx

// 通过 pkill 命令，类似于 pgrep + kill
pkill nginx

// 通过先查找再 kill 的方式
// |：管道符，管道符左边命令的输出作为管道符右边命令的输入
// grep -v grep：不包括包含grep的这个进程（因为 grep nginx 会起一个 grep 的进程）
// awk '{print $2}'：输出参数的第二个域（第二列，也就是 pid 那列）
// xargs：给后面命令传递参数的一个过滤器，这里将前面的输出变成一行
ps -ef | grep nginx | grep -v grep | awk '{print $2}' | xargs kill -9

// 除了用 awk 取第二个域之外，也可以用 cut -c 来截取指定位置的字符串（-c 表示以字符为单位）
ps -ef | grep nginx | grep -v grep | cut -c 11-15 | xargs kill -9

// 通过 pgrep 指令，根据名字找出所有包含该名字的进程号
pgrep nginx | xargs kill -9

// 通过 pidof 指令，根据进程全名找出进程号
// 与 pgrep 不同的是：第一，它需要全名；第二，它给出的结果是一行，如 5209 1230，所以不需要 xargs
pidof nginx | kill -9

// 除了管道符的方式，也可以用命令替换，这样就不用通过 xargs 转换参数了
kill -9 `pgrep nginx`
