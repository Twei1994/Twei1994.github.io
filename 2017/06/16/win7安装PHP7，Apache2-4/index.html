win7安装PHP7，Apache2.4

前言

PHP7发布有一段时间了，性能要比5.6好得多，再加上新出来的PHP-ML要求PHP的版本要至少为7.0，所以就决定再装一下PHP7。

PHP5.6和PHP7的性能对比（下图来自鸟哥的博客）

参考

本次安装主要参考了以下两篇博客：

  1. PHP7运行环境搭建（Windows7）
  2. Win7安装php7 + apache2.4，成功启动

步骤

安装前的检查

Windows版本的要求

至少得是win7(SP1)以上，也就是说win7系统，必须确保安装了SP1这个更新包，如图：

下载Visual C++ 2015

由于PHP7使用的是VC14的编译环境，所以我们还需要下载Visual C++ 2015，下载地址：https://www.microsoft.com/en-us/download/details.aspx?id=52685，根据操作系统的位数下载对应版本并安装。

下载PHP7和Apache2.4

PHP7

首先下载PHP7的windows压缩包，到这里下载http://windows.php.net/download/

要选择下载线程安全版(Thread Safe)，否则，将无法加载php7apache2_4.dll扩展；
VC14表示它是VC14编译的，需要安装Visual C++ 2015运行时环境，同时也要求Apache的版本至少为2.4

Apache2.4

接下来下载Apache2.4，下载地址为https://www.apachelounge.com/download/

开始安装

解压文件

解压下载下来的PHP7（如php-7.1.6-Win32-VC14-x64.zip）和Apache2.4（如httpd-2.4.25-win64-VC14.zip），放置在目标路径下，如我的路径是这样的：

整个web开发工具和项目都放置在E:\web\

各个版本的Apache放置在E:\web\Apache\，刚才解压的Apache2.4放在该文件夹下的Apache24中

各个版本的PHP放置在E:\web\PHP\，刚才解压的PHP7放在该文件夹下的php-7.1.6中

E:\web\www\是存放项目的根目录，也就是Apache配置中的DocumentRoot

配置Apache

打开Apache的配置文件，位于E:/web/Apache/Apache24/conf/httpd.conf

1.设置apache的安装目录

大约在配置文件的 #37 行附近。

ServerRoot "c:/Apache24" 修改为 ServerRoot "E:/web/Apache/Apache24"

2.配置工作空间

工作空间就是我们的网站根目录，大约在配置文件的 #245 行附近。

DocumentRoot "c:/Apache24/htdocs"
<Directory "c:/Apache24/htdocs">

修改为

DocumentRoot "E:/web/www"
<Directory "E:/web/www">

3.装载php模块

大约在配置文件的 #393 行附近。

在<IfModule mime_module>标签的末尾处，添加如下三行内容：

PHPIniDir "E:/web/PHP/php-7.1.6"
LoadModule php7_module "E:/web/PHP/php-7.1.6/php7apache2_4.dll"
AddType application/x-httpd-php .php

注意：此处用的是绝对路径，也可以用相对路径，相对路径的根目录是 ServerRoot 参数的值，也就是前面 ServerRoot 定义的值，并不是httpd.conf配置文件所在的目录。

4.禁止访问网站目录

大约在配置文件的 #259 行附近。

出于安全方面的考虑，最好配置该选项，禁止访问网站的目录列表。配置后只能访问的文件，不能打开目录列表。

如下：

在<Directory "E:/web/www">标签内，将Options Indexes FollowSymLinks修改为Options None

5.配置默认首页

大约在配置文件的 #280 行附近。

在<IfModule dir_module>标签内，将DirectoryIndex index.html修改为DirectoryIndex index.html index.php

配置PHP

复制E:/web/PHP/PHP-7.1.6下的php.ini-development，粘贴为php.ini。

打开该配置文件，做如下修改：

1.修改扩展目录

大约在配置文件的 #738 行附近。

extension_dir = "E:/web/PHP/php-7.1.6/ext",去掉分号

此处必须为绝对路径

2.开启相关扩展

在配置文件的 #886 行 Windows Extensions 下。

根据需要开启相关扩展，常用的有：

extension=php_gd2.dll
extension=php_curl.dll
extension=php_mbstring.dll
extension=php_openssl.dll
extension=php_mysqli.dll
extension=php_pdo_mysql.dll

3.设置时区

大约在配置文件的 #939行附近。

date.timezone ="Asia/Shanghai"

4.设置错误报告等级

在配置文件的 #460 行，可以根据需求设置。

error_reporting = E_ALL

5.可以根据自己的要求修改配置，如session和上传文件的支持等

修改上传目录（配置文件的 #820 行）：

upload_tmp_dir = "E:/web/PHP/php-7.1.6/tmp/upload"

修改session目录（配置文件的 #1355 行）：

session.save_path = ""E:/web/PHP/php-7.1.6/tmp/session

注意：tmp文件夹和它下面的两个子文件夹upload、session都需要自己新建。

开启Apache服务

把Apache加入windows服务中，并启动

进入cmd命令行，在E:\web\Apache\Apache24\bin下输入

httpd -k install

httpd -k start

这样就启动了Apache服务。

另外，重启Apache服务的命令是 httpd -k restart。

如果出现 (OS 5)拒绝访问。 : AH00369: Failed to open the WinNT service manager... 的问题，是由于权限不够造成的，以管理员身份运行cmd.exe即可安装成功。

当然，在这之前，可以先检查下刚才修改的Apache配置是不是有错，输入

httpd -t

如果返回为Syntax OK，说明没有问题。

如果出现 Starting httpd: AH00558: httpd: Could not reliably determine the... 的问题，是由于没有设置ServerName的可用域名造成的，修改成

ServerName localhost:80

测试

在E:\web\www下新建phpinfo.php文件，然后在浏览器中访问http://127.0.0.1/phpinfo.php，可以看到如下界面，则说明配置成功。

phpinfo.php 文件内容：

<?php phpinfo(); ?>

2017-7-6 更新，Mysql安装

下载

根据操作系统和操作系统位数下载 MySQL 安装压缩包，这里我下载的版本是5.7.18

地址：http://dev.mysql.com/downloads/mysql/

安装

安装也就是解压，移动到指定目录，我这里是E:\web\MySQL\，并修改文件名称为mysql-5.7.18。

配置

添加到环境变量

将mysql添加至环境变量PATH中，这样就可以在任意位置启动mysql了。

PATH的路径在：计算机 –> 系统属性 –> 高级系统配置 –> 环境变量

添加的是mysql bin文件夹的路径，例如我的是E:\web\MySQL\mysql-5.7.18\bin

初始化

在MySQL5.7.18版本之后，不在解压包里包含 my-default.ini 文件并且需要手动初始化。

As of MySQL 5.7.18, my-default.ini is no longer included in or installed by distribution packages.
Initializing the Data Directory Manually Using mysqld.

首先，以管理员身份运行cmd，进入E:\web\MySQL\mysql-5.7.18\bin目录，使用 mysqld --initialize-insecure 初始化mysql。

加 -insecure 是为了不用随机密码初始化。

初始化之后，在当前文件夹下生成了data文件夹。

添加到Windows服务中并启动

接着，输入 mysqld -install 安装服务， net start mysql 启动服务。

这样，就可以通过 mysql -u root -p 登陆mysql了。

另外， net stop mysql 是关闭MySQL服务，在windows下，重启MySQL服务需要先关闭再开启。

修改root用户密码

登陆 mysql 之后，输入set password for root@localhost = "your password";修改 root 的密码。

这样，MySQL就安装好了。
