PHP使用PDO访问数据库

前言

PDO，PHP Data Objects的简称，是PHP访问数据库的抽象层，可以看作是一个连接数据库的接口，是PHP5之后推荐使用的连接数据库的方式。

正文

它有以下几个优点：

  1. 不管是哪种数据库，都可以通过这个接口来连接和操作数据，更换时只需要更换DSN（Data Source Name）即可（有点适配器模式那个意思）；
  2. 可以有效防止SQL注入（预编译模式）；

测试数据库内容如下：数据库名称为pdo_test，数据表名称为user。

mysql> select * from user;
+----+-------+-------+
| id | name  | score |
+----+-------+-------+
|  1 | Twei  |    90 |
|  2 | Alice |    90 |
|  3 | Bob   |    80 |
|  4 | Tony  |    70 |
|  5 | Steve |   100 |
+----+-------+-------+
5 rows in set (0.00 sec)

基本操作

连接数据库

$dsn = 'mysql:host=localhost;dbname=pdo_test';
$user = 'root';
$pass = 'yourownpassword';

try {
    $dbh = new PDO($dns, $user, $pass);
}catch(PDOException $e) {
    echo 'Connection failed: ',$e->getMessage();
    die();
}

查询操作

有两种方式进行查询，分别是query（直接查询方式）和prepare + execute（预编译和参数绑定方式），第一种方式快捷方便，但存在SQL注入风险，所以建议使用第二种。

预编译方式不仅防止了SQL注入，并且在插入或修改大量数据时，因为相关的SQL已经被执行过，所以直接读取即可，速度快。

query方式:

$dbh->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
$name = 'Twei';
$sql = 'SELECT * FROM user WHERE name ='.$dbh->quote($name);
foreach ($dbh->query($sql) as $row) {
    echo $row['id'],"\t",$row['name'],"\t",$row['score'],"\n";
}

setAttribute用于设置数据库句柄属性，如错误模式等；

quote是对参数（一般为用户输入）进行转义，防止SQL注入；

以上代码输出：

1    Twei    90

prepare + execute方式：

$sql = 'SELECT * FROM user WHERE name =:name';
$stmt = $dbh->prepare($sql);
$stmt->bindParam(':name', $name, PDO::PARAM_STR);
$stmt->execute();
while ($row = $stmt->fetch(PDO::FETCH_ASSOC)) {
    echo $row['id'],"\t",$row['name'],"\t",$row['score'],"<br>";
}

在SQL语句中，通过:name占位符来代替实际变量，然后在bindParam时将该占位符替换成实际的值，通过这种方式避免SQL注入。

$dbh->prepare($sql)返回一个PDOStatement对象，之后都是通过这个对象来进行操作。

PDO::FETCH_ASSOC指定获取方式，将对应结果集中的每一行作为一个由列名索引的数组返回。如果想返回对象，可以使用PDO::FETCH_LAZY或者调用fetchObject方法。

插入数据

插入数据也是采用prepare + execute的方式，如果有多个参数，需要进行多次参数绑定。

$name = 'Logan';
$score = 93;
$sql = 'INSERT INTO user (name, score) VALUES (:name, :score)';
$stmt = $dbh->prepare($sql);
$stmt->bindParam(':name', $name, PDO::PARAM_STR);
$stmt->bindParam(':score', $score, PDO::PARAM_INT);
$stmt->execute();

执行之后数据库中数据如下所示：

mysql> select * from user;
+----+-------+-------+
| id | name  | score |
+----+-------+-------+
|  1 | Twei  |    90 |
|  2 | Alice |    90 |
|  3 | Bob   |    80 |
|  4 | Tony  |    70 |
|  5 | Steve |   100 |
|  6 | Logan |    93 |
+----+-------+-------+
6 rows in set (0.00 sec)

修改或删除数据

修改或删除数据也是采用prepare + execute的方式，如果有多个参数，需要进行多次参数绑定。

$name = 'Logan';
$score = 95;
$dbh->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
$sql = 'UPDATE user SET score = :score WHERE name = :name';
$stmt = $dbh->prepare($sql);
$stmt->bindParam(':name', $name, PDO::PARAM_STR);
$stmt->bindParam(':score', $score, PDO::PARAM_INT);
$stmt->execute();

执行之后数据库中数据如下所示：

mysql> select * from user;
+----+-------+-------+
| id | name  | score |
+----+-------+-------+
|  1 | Twei  |    90 |
|  2 | Alice |    90 |
|  3 | Bob   |    80 |
|  4 | Tony  |    70 |
|  5 | Steve |   100 |
|  6 | Logan |    95 |
+----+-------+-------+
6 rows in set (0.00 sec)

其实，通过预编译加上参数绑定方式进行增删改查，就是SQL语句不同，其余的都一样。

受影响的行数

通过Statement的rowCount方法可以返回上一次操作影响到行数。

$score = 20;
$sql = 'SELECT * FROM user WHERE score > :score';
$stmt = $dbh->prepare($sql);
$stmt->bindParam(':score', $score, PDO::PARAM_INT);
$stmt->execute();
echo $stmt->rowCount();

事务支持

PDO也支持MySQL事务（transaction），包括start、commit和rollback等。

try{
    $dbh->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
    $dbh->beginTransaction();    // 事务开始
    $sql = 'INSERT INTO user (name, score) VALUES (:name, :score)';
    $stmt = $dbh->prepare($sql);
    $stmt->execute(array(':name' => 'Natasha', ':score' => 100));
    $id = 1;
    $sql = 'DELETE FROM user WHERE id= :id';
    $stmt = $dbh->prepare($sql);
    $stmt->bindParam(':id', $id, PDO::PARAM_INT);
    $stmt->execute();
    $dbh->commit();    // 提交事务
}catch(PDOException $e){
    // rollBack写在发生错误时
    $dbh->rollBack();
    echo "Error occurred when execute SQL: ", $sql, "<br>";
    echo "Error info:<br>", $e->getMessage();
}

错误处理

PDO有3个错误提示等级，分别为POD::ERRMODE_SILENT（默认）、POD::ERRMODE_WARNING和POD::ERRMODE_EXCEPTION。

POD::ERRMODE_SILENT

不显示错误信息，会设置一个错误码，需要检查错误码来判断是否发生错误。

$name = 'Twei';
$sql = 'SELECT * FROM users WHERE name =:name';   // 数据表名有误
$stmt = $dbh->prepare($sql);
$stmt->bindParam(':name', $name, PDO::PARAM_STR);
$stmt->execute();
$code = $stmt->errorCode();
//var_dump($code);echo '<br>';   // 输出错误码，string(5) "42S02" 
if (empty($code)) {
    while ($row = $stmt->fetchObject()) {
        echo $row['id'],"\t",$row['name'],"\t",$row['score'],"<br>";
    }
}else{
    echo 'Error occurred when execut SQL ', $sql, '<br>';
    echo 'ErrorInfo:<br><pre>';
    var_dump($stmt->errorInfo());
}

执行结果如下：

Error occurred when execut SQL SELECT * FROM users WHERE name =:name
ErrorInfo:
array(3) {
  [0]=>
  string(5) "42S02"
  [1]=>
  int(1146)
  [2]=>
  string(36) "Table 'pdo_test.users' doesn't exist"
}

注意：这里的错误是Statement级别的，不是PHP级别，所以需要用$stmt->errorInfo()来获取错误信息。

POD::ERRMODE_WARNING

在这个模式下，出错会产生一个PHP的警告（warning），同时也会设置Statement级别的errorinfo。

$dbh->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_WARNING);
$name = "Twei";
$sql = 'SELECT * FROM users WHERE name =:name';
$stmt = $dbh->prepare($sql);
$stmt->bindParam(':name', $name, PDO::PARAM_STR);
$stmt->execute();
$code = $stmt->errorCode();
// var_dump($code);echo '<br>';
if (empty($code)) {
    while ($row = $stmt->fetchObject()) {
        echo $row['id'],"\t",$row['name'],"\t",$row['score'],"<br>";
    }
}else{
    echo 'Error occurred when execut SQL ', $sql, '<br>';
    echo 'ErrorInfo:<br><pre>';
    var_dump($stmt->errorInfo());
}

执行结果如下：

Warning: PDOStatement::execute(): SQLSTATE[42S02]: Base table or view not found: 1146 Table 'pdo_test.users' doesn't exist in D:\web\www\Algorithm\test.php on line 60
Error occurred when execut SQL SELECT * FROM users WHERE name =:name
ErrorInfo:
array(3) {
  [0]=>
  string(5) "42S02"
  [1]=>
  int(1146)
  [2]=>
  string(36) "Table 'pdo_test.users' doesn't exist"
}

与SILENT模式相比，多了一个Warning。

POD::ERRMODE_EXCEPTION

在这个模式下，出错会产生一个PHP的错误（PDOException），同时也会设置Statement级别的errorinfo。不过这个时候，我们就可以通过try{...}catch{...}的方式来捕获产生的异常。

$dbh->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION);
$name = 'Twei';
try {
    $sql = 'SELECT * FROM users WHERE name =:name';
    $stmt = $dbh->prepare($sql);
    $stmt->bindParam(':name', $name, PDO::PARAM_STR);
    $stmt->execute();
    while ($row = $stmt->fetchObject()) {
        echo $row['id'],"\t",$row['name'],"\t",$row['score'],"<br>";
    }
}catch(PDOException $e) {
    echo 'PDO Exception Caught.<br>';
    echo 'Error occurred with executed SQL: ',$sql,'<br>';
    echo 'Error: ', $e->getMessage(), '<br>';
    echo 'Code: ', $e->getCode(), '<br>';
    echo 'File: ', $e->getFile(), '<br>';
    echo 'Line: ', $e->getLine(), '<br>';
    echo 'Trace: ', $e->getTraceAsString(), '<br>';
}

执行结果：

PDO Exception Caught.
Error occurred with executed SQL: SELECT * FROM users WHERE name =:name
Error: SQLSTATE[42S02]: Base table or view not found: 1146 Table 'pdo_test.users' doesn't exist
Code: 42S02
File: D:\web\www\test.php
Line: 80
Trace: #0 D:\web\www\test.php(80): PDOStatement->execute() #1 {main}

通过try{...}catch{...}的方式，可以捕获execute时产生的异常，可以跟踪到文件和行数级别。

防止SQL注入

SQL注入就是攻击者在用户输入的地方输入SQL语句从而导致该语句在数据库中执行。避免SQL注入的准则就是：在外部数据到达数据库之前对它进行转义。

  1. query方式不能自动转义，需要手动调用$dbh->quote方法来转义；
  2. 预编译的方法基本可以避免SQL注入；

原理如下：当调用prepare()时，查询语句已经发送给了数据库服务器，此时只有占位符 发送过去，没有用户提交的数据；当调用到execute()时，用户提交过来的值才会传送给数据库，它们是分开传送的，所以不会产生SQL注入的风险。
